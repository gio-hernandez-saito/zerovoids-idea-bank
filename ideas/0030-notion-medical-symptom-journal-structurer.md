# Notion 의학 증상 일지 자동 구조화 플러그인

> 자유롭게 적은 건강 메모를 진료실에서 바로 꺼낼 수 있는 구조화된 의료 데이터로 변환해주는 Notion 플러그인

## 🎯 문제 정의

### 현재 상황
만성 질환을 앓거나 여러 증상을 추적해야 하는 사람들은 Notion, 메모 앱, 수첩 등에 건강 일지를 기록한다. 그러나 이 기록들은 자유 형식 텍스트로 흩어져 있어, 의사에게 설명하거나 시간에 따른 패턴을 파악하기 매우 어렵다. 매달 진료를 받을 때 "저번 달에 언제 어떤 증상이 있었나요?"라는 질문 앞에서 긴 메모를 뒤지게 된다.

### 고통 포인트
- **비구조화된 기록**: "오늘 머리가 좀 아팠고 밥도 잘 못먹었음"처럼 자유롭게 적은 내용은 검색·집계가 불가능하다
- **패턴 파악 불가**: 2주 전에도 같은 증상이 있었는지 직접 스크롤해서 찾아야 한다
- **진료 준비 부담**: 3개월치 건강 일지를 읽어서 요약본을 만들어 가는 것은 시간이 너무 많이 걸린다
- **약 복용 연관성 누락**: 증상이 특정 약 복용 시간 전후와 어떻게 연결되는지 수동으로 추적하기 어렵다

### 주요 대상
만성 질환 환자(당뇨, 루푸스, 크론병 등), 다이어리 형식으로 건강 기록을 관리하는 사람, 혹은 자녀나 부모의 건강 상태를 추적하는 보호자. 이미 Notion을 사용 중이며 별도 앱 설치를 원하지 않는 사람들.

## 💡 솔루션

### 핵심 개념
Notion 페이지에 자유롭게 쓴 건강 일지 텍스트를 분석하여 증상, 심각도, 발생 시간, 지속 시간, 유발 요인을 자동으로 추출한다. 추출된 데이터는 Notion 데이터베이스에 구조화된 속성(Property)으로 자동 입력되며, 진료용 요약 리포트와 주간·월간 증상 빈도 타임라인을 생성한다. 자유로운 글쓰기 습관을 바꾸지 않으면서 의료 수준의 데이터 품질을 얻을 수 있다.

### 작동 방식
1. **일지 작성**: 사용자는 평소처럼 Notion에 "오늘 오전 10시쯤부터 오른쪽 머리가 욱신거렸다. 4/10 정도. 타이레놀 먹고 2시간 뒤 나아짐. 커피를 3잔 마신 날이었음"처럼 자유롭게 적는다
2. **자동 파싱 트리거**: 플러그인이 지정 데이터베이스의 새 페이지를 감지하거나, 버튼 클릭 시 텍스트를 분석한다
3. **구조 추출**: NLP 파이프라인이 증상명, 위치, 심각도(숫자/형용사), 시간, 지속시간, 유발 요인, 완화 요인을 인식하고 추출한다
4. **데이터베이스 자동 채우기**: 추출된 정보를 Notion 데이터베이스 속성(증상 태그, 심각도 숫자, 날짜, 관련 약물 등)에 자동 입력한다
5. **리포트 생성**: 진료 전 버튼을 누르면 지난 30일간의 증상을 타임라인과 빈도표로 요약한 새 페이지를 생성한다

### 주요 기능
- **증상 엔티티 추출**: 자연어에서 신체 부위, 증상 유형, 심각도 척도를 인식 ("욱신거림", "찌르는 듯한", "7/10" 등)
- **약물-증상 연관 추적**: 복용 약물명을 인식하여 증상 발생 전후 약 복용 패턴을 연결
- **진료용 자동 요약**: "최근 30일 기준: 두통 8회(평균 심각도 5.2), 주로 오전, 이부프로펜 복용 후 평균 1.8시간 완화" 형태의 요약 자동 생성
- **패턴 알림**: 동일 증상이 3일 연속 등장하면 Notion 코멘트로 알림

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────────────┐
│   Notion 일지 페이지    │  (사용자가 자유롭게 작성)
└───────────┬─────────────┘
            │ Notion API (polling / webhook)
┌───────────▼─────────────┐
│   텍스트 파싱 엔진       │  (TypeScript + NLP)
│  - 증상 엔티티 추출      │
│  - 시간 표현 정규화      │
│  - 약물명 인식           │
└───────────┬─────────────┘
            │
┌───────────▼─────────────┐
│   구조화 데이터 모델     │
│  SymptomEntry {          │
│    symptom, severity,    │
│    location, timestamp,  │
│    duration, triggers,   │
│    medications           │
│  }                       │
└───────────┬─────────────┘
            │ Notion API (write)
┌───────────▼─────────────┐
│   Notion 데이터베이스   │  (속성 자동 채우기)
└───────────┬─────────────┘
            │
┌───────────▼─────────────┐
│   리포트 생성기          │  (진료용 요약 페이지)
└─────────────────────────┘
```

### 기술 스택 선정 이유
- **Notion API**: 기존 사용자 워크플로우를 방해하지 않고 데이터 읽기/쓰기 가능. 플러그인 형태로 배포 가능
- **TypeScript**: 의료 데이터는 타입 안전성이 중요. 증상 엔티티 스키마를 명확하게 정의할 수 있음
- **compromise.js (NLP)**: 경량 영어/한국어 NLP 라이브러리로 의존성 최소화. 서버 사이드에서 로컬 처리로 의료 데이터 외부 유출 방지
- **Node.js**: Notion API와 연동하는 백엔드 서비스로 적합

### 핵심 기술 난제
1. **한국어 증상 표현 다양성**: "욱신거리다", "지끈거리다", "찌릿하다"처럼 동일 증상에 대한 표현이 매우 다양함. 의료 동의어 사전(symptom synonym dictionary) 구축이 필요
2. **심각도 정규화**: "조금 아픔", "심하게", "4/10", "★★☆☆☆" 등 다양한 표현을 0-10 숫자 척도로 변환하는 휴리스틱 설계
3. **Notion API 폴링 vs 웹훅**: Notion 공식 웹훅이 제한적이라 효율적인 폴링 전략이 필요

## 🎨 무엇이 흥미로운가?

### 독특한 관점
기존 건강 앱들은 사용자에게 "구조화된 입력 폼"을 강요한다. 매일 20개 항목을 체크해야 하는 앱은 며칠 만에 포기하게 된다. 이 프로젝트는 반대 방향에서 접근한다. 사용자는 자유롭게 쓰고, 구조화는 나중에 자동으로 한다. "글쓰기 자유" + "데이터 품질"을 동시에 달성하는 역방향 설계.

### 혁신 포인트
- 자유 텍스트 → 의료 데이터베이스로의 자동 변환 파이프라인
- Notion을 EMR(전자의무기록) 대체제로 사용하는 비전통적 활용
- 한국어 의료 표현에 특화된 경량 NLP 처리

### 학습 기회
- Notion API의 데이터베이스 스키마 설계와 Property 시스템
- 한국어 자연어 처리: 형태소 분석, 엔티티 인식
- 의료 정보학 기초: 증상 코딩 체계(ICD-10 등) 이해
- 폴링 기반 이벤트 감지 아키텍처 설계

### 이 공백이 존재하는 이유
자유 텍스트 파싱을 의료 도메인에 적용하려면 의료 전문 용어와 NLP를 결합해야 한다. 일반 NLP 도구는 의료 표현에 약하고, 의료 전문 AI는 과도하게 복잡하다. Notion API의 쓰기 기능이 2022년부터 안정화되면서 이제 실현 가능해진 영역이다.
