# Lightroom 플러그인 — 사진 구도 분석기

> Adobe Lightroom 안에서 사진을 선택하는 순간, 삼분할법·황금비율·선 구도를 자동으로 겹쳐 보여주는 플러그인

## 🎯 문제점

### 현재 상황
Adobe Lightroom은 세계에서 가장 많이 쓰이는 사진 편집 도구지만, 내장된 구도 분석 기능은 존재하지 않는다. 사진가들은 Lightroom에서 컬링(culling)을 하면서 "이 사진의 구도가 좋은가?"를 오로지 직관에만 의존한다. 구도 가이드를 보려면 별도 앱으로 내보내거나 Photoshop을 열어야 한다.

### 불편함
- **워크플로 단절**: 구도 확인을 위해 Lightroom을 벗어나야 함 — 컬링 속도가 현저히 떨어짐
- **초보자의 학습 공백**: 어떤 사진이 "왜" 구도적으로 좋은지 피드백을 받을 수 없음
- **Lightroom SDK는 있지만 플러그인이 없다**: Lua 기반 Lightroom SDK가 공개되어 있음에도 구도 분석 플러그인은 오픈소스에서 찾기 어려움

### 누가 이 문제를 겪는가?
사진 편집 워크플로를 Lightroom에서 완결하고 싶은 사진 애호가 및 프로 포토그래퍼. 특히 대량의 사진을 빠르게 컬링해야 하는 웨딩 사진가, 여행 사진가, 사진 강의를 진행하는 강사.

## 💡 해결책

### 핵심 아이디어
Lightroom SDK(Lua)로 만든 플러그인이 현재 선택된 사진 위에 구도 오버레이를 그려준다. 삼분할법 격자, 황금비율 나선, 대각선 구도선, 중심 대칭 가이드 등을 토글 방식으로 전환하며, 각 오버레이가 피사체·지평선·주요 요소와 얼마나 일치하는지를 0~100점으로 수치화한다.

### 동작 방식
1. **플러그인 설치**: Lightroom의 플러그인 매니저에 `.lrplugin` 폴더를 등록
2. **사진 선택**: Library 또는 Develop 모듈에서 사진을 클릭하면 자동으로 분석 패널이 활성화
3. **오버레이 렌더링**: 선택한 구도 유형(삼분할, 황금비율, 대각선 등)이 사진 위에 반투명 레이어로 겹쳐짐
4. **점수 표시**: 피사체 감지 결과를 바탕으로 각 구도 규칙과의 정렬도를 점수로 표시
5. **배치 리포트**: 폴더 전체를 분석해 "구도 점수 상위 20장" 필터를 Lightroom 컬렉션으로 자동 생성

### 주요 기능
- **다중 구도 오버레이**: 삼분할법 / 황금비율 나선 / 피보나치 격자 / 대각선 / 중심 대칭을 탭으로 전환
- **구도 점수**: 엣지 감지 + 간단한 피사체 위치 추정으로 각 구도 규칙과의 일치도를 수치화
- **배치 컬링 보조**: 여러 장을 동시에 분석해 점수 기반으로 자동 플래그 처리
- **학습 모드**: 해당 구도 규칙의 설명과 유명 사진 예시를 패널에 함께 표시

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────────────────────┐
│   Lightroom (Lua SDK)           │
│  ┌─────────────────────────┐   │
│  │  Plugin UI Panel (Lua)  │   │  ← 오버레이 토글, 점수 표시
│  └───────────┬─────────────┘   │
│              │ 이미지 경로 전달  │
└──────────────┼──────────────────┘
               ▼
┌─────────────────────────────────┐
│  로컬 분석 서버 (Python/FastAPI) │
│  ┌─────────────────────────┐   │
│  │  OpenCV 엣지 감지        │   │  ← 피사체 위치 추정
│  │  구도 점수 계산기         │   │  ← 삼분할/황금비율 정렬도
│  └─────────────────────────┘   │
└──────────────┬──────────────────┘
               ▼
┌─────────────────────────────────┐
│  오버레이 렌더러 (Canvas/SVG)    │
│  → Lua가 결과를 LrView로 렌더링 │
└─────────────────────────────────┘
```

### 기술 스택 선택 이유
- **Lua (Lightroom SDK)**: Lightroom 플러그인의 유일한 공식 언어. UI 패널 생성, 현재 사진 접근, 메타데이터 수정이 가능
- **Python + OpenCV**: 이미지 분석의 핵심. 엣지 감지, 컨투어 분석, 피사체 위치 추정에 검증된 라이브러리 생태계
- **FastAPI 로컬 서버**: Lua에서 직접 이미지 처리가 어렵기 때문에 localhost HTTP 서버를 경유. 설치형이라 외부 API 의존 없음
- **TypeScript (설정 UI)**: 플러그인 설정 페이지를 별도 웹뷰로 열 경우 사용. 선택적 컴포넌트

### 핵심 기술 과제
1. **Lua ↔ Python 통신**: Lightroom Lua SDK는 HTTP 요청을 직접 지원하지 않음. `LrHttp` 모듈 또는 외부 프로세스 실행(`LrTasks`)으로 해결 가능하지만 비동기 처리가 까다로움
2. **오버레이 정확도**: 피사체 위치를 완벽히 감지하려면 ML 모델이 필요하지만, MVP에서는 엣지 밀도 기반 추정으로 80% 수준의 정확도를 확보할 수 있음
3. **성능**: RAW 파일(50MB+)을 매번 분석하면 느림. 미리 생성된 프리뷰 JPEG를 가져오는 방식으로 최적화

### 새로운 기술 및 개념
- **Lightroom Lua SDK**: 플러그인 아키텍처, `LrView`, `LrHttp`, `LrTasks` API의 동작 원리

- **이종 언어 IPC 설계**: Lua에서 Python 프로세스를 제어하는 패턴 — 일반적인 웹 개발에서는 접하기 어려운 영역
- **사진 구도 이론의 수학적 표현**: 황금비율(1.618)을 픽셀 좌표로 변환하는 방법

### 성장 기회
- "제약이 많은 환경"에서 창의적 우회 아키텍처를 설계하는 경험 — 이 사고방식은 레거시 시스템 통합에도 전이됨
- 시각적 미학을 수치로 표현하는 방법론 — 데이터 시각화 전문성과 시너지
- 플러그인 패키징·배포·업데이트 관리 (npm 생태계와는 전혀 다른 방식)

### 도전 영역
스크립트 언어인 Lua를 처음 다루면서, 동시에 이미지 분석 파이프라인을 연결하는 것. 두 가지 낯선 요소가 동시에 등장하지만, 각각의 책임이 명확히 분리되어 있어 점진적으로 해결 가능하다.
