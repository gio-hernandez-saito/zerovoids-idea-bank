# 기상 데이터로 배우는 통계 개념 인터랙티브 튜터

> 어제의 날씨 데이터를 가지고 평균, 분산, 상관관계를 직접 손으로 만지며 배우는 살아있는 통계 교과서

## 🎯 문제 정의

### 현재 상황
통계 개념은 수식만 보면 추상적이고 지루하다. 표준편차를 설명하는 교과서는 "데이터의 퍼짐 정도"라고만 쓰여 있고, 학습자는 그게 실제 세계에서 어떤 의미인지 감이 오지 않는다. 대부분의 통계 학습 도구는 가상의 숫자 배열을 쓰거나, 무미건조한 예시 데이터셋(붓꽃, 타이타닉)을 반복 사용한다.

### 고통 포인트
- **추상성의 벽**: "분산이 크다"는 말이 실생활에서 무엇을 의미하는지 이미지가 그려지지 않음
- **정적인 예시**: 교과서 속 숫자는 변하지 않아 "조작해보며 배우기"가 불가능
- **맥락 단절**: 수식을 외워도 실제 데이터에 적용하는 순간 막막함

### 누가 겪는 문제인가?
통계를 처음 배우는 개발자, 데이터 시각화를 공부하는 프론트엔드 개발자, 통계 수업을 듣는 대학생, 분석 업무를 시작하는 기획자 등 — 수식보다 시각적 직관이 먼저인 사람들.

## 💡 해결책

### 핵심 개념
매일 자동으로 업데이트되는 실제 날씨 데이터(기온, 강수량, 풍속, 습도)를 기반으로 통계 개념을 단계별로 인터랙티브하게 가르치는 웹 튜터다. 사용자는 슬라이더로 데이터 범위를 조정하고, 아웃라이어를 직접 추가/제거하며, 두 도시의 데이터를 나란히 놓고 상관관계를 체험한다. 수식이 먼저가 아니라, 조작이 먼저고 수식이 뒤따른다.

### 작동 방식
1. **도시 선택**: 서울, 도쿄, 런던 등 친숙한 도시의 최근 30일 날씨 데이터를 자동 로딩
2. **개념 챕터 탐색**: 평균 → 중앙값 → 분산/표준편차 → 상관관계 순서로 챕터를 따라가며 학습
3. **조작과 관찰**: 각 챕터에서 데이터 포인트를 드래그하거나 필터링하면 지표가 실시간으로 바뀌고, 그 의미를 텍스트로 설명
4. **퀴즈로 확인**: 챕터 말미에 "이 도시는 왜 표준편차가 더 클까요?" 형식의 서술형 퀴즈로 이해 점검

### 주요 기능
- **살아있는 데이터**: Open-Meteo API로 실제 기상 데이터를 무료로 불러와 매일 갱신
- **인터랙티브 조작**: D3.js 기반 드래그 가능한 산점도, 히스토그램 — 데이터를 직접 건드려서 수치 변화를 눈으로 확인
- **이중 도시 비교**: 두 도시를 나란히 놓고 상관관계(기온 vs 습도)를 시각적으로 탐색
- **수식 연동 하이라이트**: 조작할 때마다 수식의 어느 부분이 변하는지 하이라이트로 추적

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────────────────────────────┐
│           Open-Meteo API               │
│  (기온, 습도, 강수량, 풍속 — 무료)       │
└────────────────┬────────────────────────┘
                 │ fetch (클라이언트 직접)
┌────────────────▼────────────────────────┐
│         Data Layer (TypeScript)         │
│  - 날씨 데이터 파싱 및 정규화            │
│  - 통계 연산 함수 (순수 함수)            │
│    mean(), variance(), correlation()    │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│         Lesson Engine (React)           │
│  - 챕터 진행 상태 관리                   │
│  - 퀴즈 로직                            │
│  - 수식 하이라이트 연동                  │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│      Visualization Layer (D3.js)        │
│  - 드래그 가능한 산점도                  │
│  - 인터랙티브 히스토그램                 │
│  - 실시간 통계 오버레이                  │
└─────────────────────────────────────────┘
```

### 기술 스택 선택 이유
- **Open-Meteo API**: 무료, 인증 불필요, CORS 허용 — 튜터 앱이 백엔드 없이 동작 가능
- **D3.js**: 드래그 이벤트와 데이터 바인딩을 직접 연결하기 가장 적합한 라이브러리
- **React + TypeScript**: 챕터/퀴즈 상태 관리와 컴포넌트 재사용에 유리
- **Vite**: 빠른 개발 환경, 번들 최적화

### 주요 기술 도전
1. **인터랙티브 수식 동기화**: D3 드래그 이벤트로 데이터가 바뀔 때 MathJax/KaTeX로 렌더링된 수식의 특정 변수를 하이라이트하는 것 — DOM 이벤트와 수식 렌더링을 연결하는 설계가 필요
2. **페다고지 시퀀싱**: "조작 → 관찰 → 설명 → 수식" 순서로 학습 흐름을 강제하는 상태 머신 설계

## 🎨 무엇이 흥미로운가?

### 독특한 각도
날씨 데이터는 누구나 직관적으로 이해하는 실세계 데이터다. "서울의 1월 기온 표준편차가 도쿄보다 크다"는 말은 즉각적인 공감을 유발한다. 추상적인 숫자 배열 대신 친숙한 도메인 데이터를 쓰면 통계 개념의 진입 장벽이 극적으로 낮아진다.

### 혁신 포인트
- 매일 새 데이터가 자동 업데이트되므로 "살아있는 교과서" 효과
- 수식이 인터랙션에 반응한다 — 가르치는 방향이 반대 (경험 → 이론)
- 날씨라는 도메인이 기상학 크로스오버를 자연스럽게 달성

### 학습 기회
- D3.js의 드래그 이벤트와 데이터 바인딩 심층 이해
- KaTeX/MathJax DOM 조작과 React 통합
- 순수 함수로 통계 알고리즘 구현 (mean, variance, Pearson correlation)
- 교육용 UX 패턴 (scaffolding, 점진적 공개)

## 📊 시장 및 검증

### 유사 프로젝트
- **Seeing Theory** (seeing-theory.brown.edu): 아름다운 통계 시각화. 하지만 정적이고, 실제 데이터가 아닌 가상 분포 사용
- **StatQuest** (YouTube): 훌륭한 설명이지만 비디오 형식이라 직접 조작 불가
- **Observable notebooks**: 전문가용, 코딩 없이 배우기 어려움
