# 경제 위기 전파 네트워크 시뮬레이터

> 2008년 리먼 브라더스 붕괴처럼, 한 노드의 파산이 전체 금융 네트워크를 어떻게 무너뜨리는지 직접 조작하며 관찰한다.

## 🎯 문제 정의

### 현재 상황
경제 위기의 전파(contagion) 개념은 경제학 교과서와 논문에서 텍스트와 정적 다이어그램으로만 설명된다. "상호 연결된 금융 기관들이 연쇄 부도를 일으킨다"는 문장을 읽어도, 실제로 어떤 조건에서 전파가 멈추고 어떤 조건에서 시스템 전체가 붕괴하는지 직관적으로 이해하기 어렵다.

### 핵심 불편함
- **추상적 설명의 한계**: 네트워크 밀도, 레버리지 비율, 중앙성(centrality)이 위기 전파에 어떤 영향을 미치는지 글로는 체감 불가
- **인과관계 불투명**: "왜 2008년엔 전체가 붕괴했고, 1997년 아시아 위기는 일부 국가에 머물렀나?" — 파라미터 차이를 직접 조작해보지 않으면 모름
- **학습 도구 부재**: 경제학도나 데이터 사이언티스트가 전파 모델을 실험할 수 있는 인터랙티브 샌드박스가 없음

### 이 문제를 겪는 사람
경제학을 독학하는 개발자, 금융 데이터 분석을 배우는 학생, 복잡계 네트워크 이론에 관심 있는 엔지니어 — 개념은 알지만 파라미터와 결과의 관계를 손으로 만져보고 싶은 사람.

## 💡 해결책

### 핵심 개념
경제 위기 전파 네트워크 시뮬레이터는 금융 기관들을 노드로, 대출·투자 관계를 엣지로 표현하는 그래프 위에서 전파 시뮬레이션을 실행하는 도구다. 사용자는 네트워크 구조(랜덤, 스케일-프리, 소세계), 각 노드의 자본 비율(leverage), 충격의 크기와 진원지를 직접 설정하고 "전파 시작" 버튼을 누른다. 시뮬레이션은 스텝-바이-스텝으로 진행되며, 어떤 기관이 언제 도산하고 손실이 어떻게 이웃으로 퍼지는지 실시간으로 애니메이션된다.

### 작동 방식
1. **네트워크 구성**: 슬라이더로 노드 수(10~200), 엣지 밀도, 네트워크 토폴로지(Erdős–Rényi / Barabási–Albert / Watts–Strogatz)를 선택
2. **파라미터 설정**: 각 노드의 초기 자본 비율, 외부 충격 크기(shock magnitude), 전파 임계값(contagion threshold) 조정
3. **시뮬레이션 실행**: 선택한 노드에 충격 주입 → 이웃 노드로 손실 전파 → 도산 조건 달성 시 연쇄 반응
4. **결과 분석**: 최종 도산 노드 수, 전파 속도, 시스템 생존율, 가장 취약한 노드(betweenness centrality 기준) 하이라이트

### 주요 기능
- **토폴로지 비교 모드**: 동일한 충격을 서로 다른 네트워크 구조에 동시 적용해 결과 비교
- **취약 노드 식별**: 어떤 노드를 제거하면 전파가 멈추는지 역산하는 "백신 접종(immunization)" 모드
- **역사적 시나리오 프리셋**: 2008 글로벌 금융위기, 1997 아시아 외환위기 파라미터 근사값 제공
- **전파 속도 조절**: 슬로우 모션으로 각 스텝 관찰 가능

## 🏗️ 기술 아키텍처

### 시스템 구조

```
┌─────────────────────────────────────┐
│           Svelte UI Layer           │
│  (슬라이더, 컨트롤, 통계 패널)         │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│       Simulation Engine (TS)        │
│  NetworkGraph 모델 + ContagionModel  │
│  (노드 상태, 전파 규칙, 스텝 계산)      │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│        D3.js Renderer               │
│  Force-directed layout + 애니메이션   │
│  (노드 색상: 건전/위험/파산 상태 표현)  │
└─────────────────────────────────────┘
```

### 기술 스택 선택 이유
- **Svelte**: 반응형 파라미터 바인딩이 React보다 훨씬 간결. 슬라이더 → 시뮬레이션 파라미터 → 재계산의 반응형 체인에 최적. Weekly constraint(No React) 충족.
- **D3.js**: Force-directed graph 레이아웃과 트랜지션 애니메이션을 위한 최선의 선택. 노드 상태 변화를 색상으로 인코딩하는 데 강력함.
- **TypeScript**: 네트워크 모델(노드/엣지 타입), 전파 알고리즘의 타입 안정성 확보.

### 핵심 기술 도전
1. **전파 알고리즘 설계**: 단순 SIR 모델이 아닌 금융 특화 전파 — 손실의 비례적 분배, 자본 비율에 따른 임계값 계산. DebtRank 알고리즘 참조 구현 필요.
2. **그래프 레이아웃과 애니메이션 동기화**: D3 force simulation이 진행 중일 때 노드 상태가 스텝별로 변하면 레이아웃이 흔들릴 수 있음. 레이아웃을 안정화 후 고정(freeze)한 뒤 상태 애니메이션만 실행하는 분리 전략 필요.
3. **대규모 네트워크 성능**: 200개 노드 × 엣지 × 애니메이션 프레임. SVG 대신 Canvas를 D3와 함께 사용하는 hybrid 렌더링 고려.

## 🎨 왜 흥미로운가?

### 독창적인 접근
기존 경제학 시뮬레이션 도구들은 대부분 학술 Python 노트북(NetworkX + matplotlib) 형태로, 인터랙티브하지 않고 설치 장벽이 높다. 웹 기반으로 "조작 즉시 결과 관찰" 경험을 제공하는 도구는 드물다. 경제 위기라는 무거운 주제를 게임처럼 탐험할 수 있게 만드는 것이 핵심 차별점이다.

### 혁신 포인트
- 경제학의 전파 이론(contagion theory)과 그래프 시각화의 크로스도메인 결합
- 정적 교과서 개념을 파라미터 조작 가능한 샌드박스로 변환
- "어떤 노드를 구해야 시스템이 살아남는가" — 게임 이론적 사고를 시각화로 체험

### 학습 기회
- D3.js force-directed graph의 심층 이해 (레이아웃 안정화, 커스텀 힘 함수)
- 그래프 이론 알고리즘 (betweenness centrality, degree distribution)
- DebtRank / 전파 모델링 (금융 복잡계 이론)
- Svelte의 반응형 스토어 패턴으로 복잡한 상태 관리
