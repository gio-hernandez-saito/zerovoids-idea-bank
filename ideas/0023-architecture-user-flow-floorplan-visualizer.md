# 건축 동선 분석기 — 웹사이트 사용자 흐름을 건물 평면도로

> 사용자가 어떻게 당신의 웹사이트를 "걷는지" 건축 평면도처럼 한눈에 파악하라 — 막힌 복도, 막힌 출구, 접근하기 어려운 방까지.

## 🎯 문제

### 현재 상황
현재 UX 분석 도구들은 사용자 흐름을 추상적인 화살표 다이어그램이나 숫자로 보여준다. GA4의 퍼널, Hotjar의 히트맵, Mixpanel의 플로우 — 모두 유용하지만 공간적 직관이 없다. 건축가가 사람들이 건물을 어떻게 이동하는지 수십 년째 연구해온 동선(circulation) 이론이 있음에도, 디지털 공간에는 이 언어가 없다.

특히 접근성 관점에서, 어떤 페이지가 특정 사용자 그룹(스크린리더 사용자, 키보드 사용자)에게 "막힌 방"처럼 작동하는지 시각적으로 드러내는 도구는 존재하지 않는다.

### 주요 고통 포인트
- **동선 단절 탐지 불가**: 어떤 페이지가 "막다른 복도"처럼 이탈을 유발하는지 흐름으로 볼 수 없다
- **접근성 병목 비가시화**: 키보드 내비게이션이나 스크린리더 사용자가 어디서 "문을 못 여는지" 데이터가 없다
- **비기술자와의 소통 장벽**: 개발자가 퍼널 데이터를 디자이너나 기획자에게 설명할 때 추상적 숫자만 전달된다

### 누가 이 문제를 겪는가?
UX 리서처, 프로덕트 매니저, 접근성 담당자, 그리고 자신의 웹사이트 구조를 직관적으로 이해하고 싶은 1인 개발자 및 소규모 팀.

## 💡 해결책

### 핵심 개념
건축 동선 분석기는 웹사이트의 페이지 구조와 사용자 이동 패턴을 **건물 평면도 메타포**로 변환한다. 각 페이지는 방(room), 링크는 문(door), 사용자 흐름의 밀도는 복도 너비, 이탈이 많은 지점은 막힌 벽으로 표현된다. 여기에 접근성 데이터를 오버레이하면 "이 방에는 휠체어(스크린리더)가 들어갈 수 없다"는 정보를 평면도 위에 직관적으로 표시한다.

사용자는 sitemap XML 또는 URL 목록을 입력하면, 도구가 각 페이지를 크롤링해 링크 구조를 파악하고, 선택적으로 실제 분석 데이터(GA CSV 내보내기)를 붙여 넣어 동선을 시각화한다.

### 동작 방식
1. **입력**: sitemap.xml URL 혹은 직접 URL 목록 붙여넣기 — 도구가 링크 관계를 자동 파싱
2. **평면도 생성**: D3.js 기반 레이아웃 알고리즘이 페이지를 방으로, 링크를 문으로 배치
3. **트래픽 오버레이**: GA4 CSV(페이지뷰, 이탈률)를 드래그앤드롭하면 복도 너비와 방 크기가 유입량에 비례
4. **접근성 스캔**: 선택적으로 각 URL의 기본 접근성 체크(axe-core 기반) 실행 — 실패한 방은 빨간 테두리 표시
5. **동선 분석**: "막힌 복도"(이탈율 80% 이상), "미로"(3단계 이상 깊이), "접근 불가 방"(접근성 위반)을 자동 하이라이트

### 핵심 기능
- **건축 평면도 레이아웃**: 단순 플로우차트가 아니라 공간적 평면도 스타일 — 방, 복도, 문, 계단(depth level)으로 구성
- **접근성 열지도**: 각 페이지의 WCAG 위반 심각도를 방 색상으로 표현 (초록→노랑→빨강)
- **동선 병목 자동 감지**: 모든 경로가 반드시 통과하는 "복도 페이지"를 강조 — 이 페이지가 느리거나 접근 불가이면 전체 사이트가 영향받음
- **비교 모드**: 리디자인 전후 구조를 두 평면도로 나란히 비교
- **내보내기**: 팀 공유용 PNG/SVG 내보내기, 또는 인터랙티브 HTML 리포트

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────────┐
│   입력 레이어       │  sitemap.xml / URL 목록 / GA CSV
└─────────┬───────────┘
          │
┌─────────▼───────────┐
│   파서 & 크롤러     │  link-graph 추출, axe-core 접근성 체크
└─────────┬───────────┘
          │
┌─────────▼───────────┐
│   그래프 모델       │  계층 구조 + 트래픽 가중치 + 접근성 점수
└─────────┬───────────┘
          │
┌─────────▼───────────┐
│   평면도 레이아웃   │  D3.js treemap/force + 건축 메타포 렌더링
└─────────┬───────────┘
          │
┌─────────▼───────────┐
│   React UI          │  인터랙션, 오버레이 토글, 내보내기
└─────────────────────┘
```

### 기술 스택 선정 이유
- **D3.js**: 커스텀 레이아웃 알고리즘 구현에 필수 — 기성 차트 라이브러리로는 평면도 메타포를 구현 불가. force simulation + 커스텀 rectangle packing으로 방 배치
- **axe-core**: 브라우저 내 접근성 감사 표준 라이브러리 — 각 URL을 iframe으로 로드 후 axe 실행
- **React + TypeScript**: 복잡한 상태(URL 목록, 분석 결과, 레이아웃 설정)를 타입 안전하게 관리
- **Vite**: 빠른 개발 환경, 라이브러리 모드로 npm 패키지 배포 가능

### 핵심 기술 도전
1. **건축 레이아웃 알고리즘**: 페이지를 의미 있는 공간으로 배치하는 것이 어렵다. 단순 force-directed layout은 건축 평면도 느낌이 나지 않는다. Squarified Treemap을 기반으로 하되, 링크 관계를 "문" 위치로 표현하기 위해 커스텀 edge routing이 필요하다.
2. **교차 출처 axe-core 실행**: 외부 URL을 iframe으로 로드해 axe-core를 실행하면 CORS 제약이 있다. 브라우저 확장 방식 또는 puppeteer 기반 Node.js 백엔드로 우회 필요 — MVP에서는 동일 출처 페이지만 지원하고 이후 확장.
3. **대형 사이트 성능**: 1000개 이상의 페이지를 평면도로 렌더링하면 DOM이 폭발한다. LOD(Level of Detail) 시스템으로 zoom out 시 섹션을 그룹으로 collapse.

## 🎨 무엇이 흥미로운가?

### 독특한 관점
기존 UX 도구들은 "데이터"를 보여준다. 이 도구는 "공간"을 보여준다. 건축가가 수십 년간 연구한 동선 이론 — 주동선/부동선, 병목 공간, 접근성 — 을 디지털 제품에 적용한 시도는 거의 없다. 특히 접근성을 "방에 들어갈 수 있는가"로 표현하면, 비기술자도 즉각적으로 이해한다.

### 혁신 포인트
- 건축 동선 이론(Circulation Theory)을 UX 분석에 최초 적용
- 접근성 데이터를 공간 은유로 표현 — 숫자가 아니라 "막힌 문"으로
- D3.js로 완전히 커스텀 평면도 레이아웃 엔진 구현

### 학습 기회
- D3.js 고급 레이아웃 알고리즘 (Treemap, Force Simulation 결합)
- axe-core API와 WCAG 접근성 규칙 내부 구조
- 그래프 이론 (계층 그래프, 최단 경로, 중심성)
- 공간 은유 기반 정보 시각화 이론

### 이 공백이 존재하는 이유
건축 동선 이론은 UX 학계에서 논의되지만 실제 도구로 구현된 사례가 없다. 기존 분석 도구 개발자들이 건축 메타포보다 익숙한 플로우차트를 선택했기 때문이다. 접근성 + 사이트 구조 + 시각적 동선을 한 도구로 묶은 것도 없다.

## 🌱 배울 것들

### 새로운 기술과 개념
- **D3.js 고급 레이아웃**: Treemap, Force Simulation, 커스텀 Rectangle Packing을 결합해 의미 있는 공간 배치 구현

- **axe-core 내부 구조**: WCAG 규칙이 어떻게 DOM을 검사하는지 이해하고, 어떤 위반이 실제로 심각한지 판단하는 법
- **공간 은유 시각화**: 추상 데이터를 물리적 공간으로 매핑하는 정보 시각화 이론

### 성장 기회
- 복잡한 그래프 레이아웃 알고리즘을 직접 구현하면서 "왜 force-directed가 모든 상황에 맞지 않는가"를 체감
- 접근성을 코드 레벨이 아닌 시스템 레벨(사이트 전체 흐름)에서 분석하는 시각 획득
- 비기술자와의 데이터 소통 문제를 은유(metaphor) 설계로 해결하는 UX 글쓰기

### 도전 구역
평면도 레이아웃 알고리즘 — 기성 D3 레이아웃을 넘어 건축 도면처럼 보이게 만드는 커스텀 배치 로직은 순수한 기술적 창의력을 요구한다.
