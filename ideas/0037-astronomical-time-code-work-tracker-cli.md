# 천문 시간 기반 코드 작업 타임트래커 CLI

> 태양, 달, 별의 주기로 하루 코딩 흐름을 추적하는 터미널 전용 시간 기록 도구

## 🎯 문제 정의

### 현재 상황
일반적인 타임트래커는 시간을 단순한 숫자(09:00~11:30)로만 기록한다. 하지만 개발자의 집중력은 하루 중 시간대에 따라 크게 달라진다. 새벽 2시의 코딩과 오후 2시의 코딩은 질적으로 전혀 다른 작업이지만, 기존 도구는 이를 동일하게 "2시간"으로만 기록한다.

### 페인 포인트
- **맥락 없는 시간 기록**: "어제 총 6시간 코딩했다"는 기록이 실제 생산성을 전혀 반영하지 못함
- **집중력 패턴 파악 불가**: 내가 언제 가장 깊은 집중 상태에 들어가는지 데이터가 없음
- **단조로운 로그**: 숫자만 가득한 타임시트는 회고할 동기를 주지 못함

### 누가 이 문제를 겪는가?
자유로운 시간대에 일하는 프리랜서 개발자, 야간 코딩을 즐기는 사이드 프로젝트 개발자, 그리고 자신의 생산성 패턴을 이해하고 싶은 모든 개발자.

## 💡 해결책

### 핵심 개념
천문 시간 기반 코드 작업 타임트래커 CLI는 작업 시간을 기록할 때 해당 순간의 천문학적 상태(일출/일몰까지 남은 시간, 달의 위상, 황혼 단계)를 함께 저장한다. 단순한 시계 시간 대신 "오늘 달이 초승달인 밤 11시에 시작해서 새벽 2시까지 집중했다"는 식의 의미 있는 맥락이 쌓인다. 축적된 데이터를 통해 자신의 집중력이 어떤 천문학적 환경에서 가장 높은지 패턴을 발견할 수 있다.

### 동작 방식
1. **작업 시작**: `astro-track start "컴포넌트 설계"` — 현재 위치 기반으로 일출/일몰 상태, 달의 위상, 황혼 단계를 자동 계산해 세션에 저장
2. **작업 종료**: `astro-track stop` — 세션 종료, 소요 시간과 함께 작업 중 달라진 천문 상태 변화도 기록
3. **패턴 조회**: `astro-track report --by phase` — 달의 위상별, 황혼 단계별, 태양 고도별로 집중 시간 분포를 터미널 차트로 출력

### 핵심 기능
- **천문 상태 자동 감지**: 사용자 위치(위도/경도) 기반으로 일출·일몰·달의 위상·황혼 유형(시민/항해/천문 황혼)을 계산
- **세션 태깅**: 작업 유형(설계, 구현, 디버깅, 코드리뷰)을 태그로 기록하여 어떤 종류의 작업이 어떤 시간대에 잘 되는지 분석
- **터미널 바 차트**: `report` 명령으로 ASCII 막대 그래프로 패턴 시각화 (UI 없이 터미널에서만 동작)
- **JSON/CSV 내보내기**: 원시 데이터 내보내기로 다른 분석 도구와 연동 가능

## 🏗️ 기술 아키텍처

### 시스템 구조

```
┌──────────────────────────────────────┐
│         CLI 레이어 (Commander.js)     │
│  start | stop | status | report       │
└─────────────────┬────────────────────┘
                  │
       ┌──────────▼──────────┐
       │   세션 매니저        │
       │  (시작/종료/조회)    │
       └──────────┬──────────┘
          ┌───────┴────────┐
          ▼                ▼
┌──────────────┐  ┌──────────────────┐
│  천문 계산기  │  │  로컬 스토리지   │
│ (astronomia) │  │  (~/.astro-track/ │
│ 위상/고도/   │  │   sessions.json) │
│ 황혼 계산    │  └──────────────────┘
└──────────────┘
          │
┌──────────▼──────────┐
│   리포트 렌더러       │
│  (ASCII 차트 출력)   │
└─────────────────────┘
```

### 기술 스택 선택 이유
- **TypeScript**: 타입 안전성 확보, 천문 계산 결과를 명확하게 모델링
- **Commander.js**: CLI 서브커맨드와 옵션 파싱의 표준 라이브러리
- **astronomia**: 순수 JS 천문학 계산 라이브러리 — 외부 API 없이 오프라인으로 달의 위상, 태양 고도, 황혼 단계 계산 가능
- **로컬 JSON 스토리지**: 외부 DB 없이 `~/.astro-track/` 디렉토리에 세션 저장

### 핵심 기술 챌린지
1. **정확한 황혼 계산**: 시민 황혼(태양 고도 -6°), 항해 황혼(-12°), 천문 황혼(-18°)을 위도/경도 기반으로 정확히 계산해야 함. `astronomia` 라이브러리의 `solar` 모듈 활용
2. **위치 정보 관리**: GPS 없이 CLI에서 위치를 어떻게 얻을 것인가. 최초 설정 시 `astro-track config --lat 37.5 --lon 127.0`으로 저장하거나 IP 기반 대략적 위치 추정 방식 중 선택

### 혁신 포인트

- "나는 초승달 밤에 집중이 잘 된다"는 패턴을 데이터로 증명하거나 반증하는 개인화된 인사이트

### 학습 기회
- `astronomia` 라이브러리를 통한 구면 천문학 기초 (황도 좌표계, 태양 고도각 계산)

- **Watson CLI** (github.com/TailorDev/Watson): CLI 타임트래커. 시간을 단순 숫자로만 기록, 환경 맥락 전무
- **Timewarrior** (timewarrior.net): 터미널 타임트래커. 강력하지만 천문 데이터와 무관

### 왜 이 공백이 존재하는가?
타임트래킹 도구는 "업무 보고용" 관점에서 설계됐기 때문에 숫자(시간, 요금)만 중요시했다. 개인 생산성 패턴 탐구라는 관점은 상대적으로 소외됐고, 천문학을 개발 도구에 결합한다는 발상 자체가 매우 틈새적이다.

### 성공 지표
- [ ] `start`, `stop`, `report` 3개 핵심 커맨드 동작
- [ ] 달의 위상 8단계 정확히 계산
- [ ] 100개 세션 데이터로 패턴 리포트 생성 가능
- [ ] `npm install -g astro-track`으로 설치 즉시 사용 가능
