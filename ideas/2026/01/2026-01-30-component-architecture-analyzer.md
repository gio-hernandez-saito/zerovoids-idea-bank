# 컴포넌트 아키텍처 분석기

> React 컴포넌트의 아키텍처 품질과 패턴 준수도를 한눈에 분석하고 개선점을 제안하는 도구

## 🎯 문제 상황

### 현재 상황
React 프로젝트가 커질수록 컴포넌트 구조가 복잡해지고, 아키텍처 원칙을 지키기 어려워집니다. 개발자들은 "이 컴포넌트가 너무 많은 책임을 가지고 있나?", "props drilling이 심한가?", "재사용 가능한 컴포넌트인가?" 같은 질문에 명확한 답을 얻기 어렵습니다.

### 핵심 문제점
- **아키텍처 품질 가시성 부족**: 컴포넌트 구조의 문제점을 객관적으로 측정하기 어려움
- **패턴 준수 체크의 어려움**: Atomic Design, FSD 등의 아키텍처 패턴을 제대로 따르고 있는지 확인하기 어려움
- **리팩토링 우선순위 결정**: 어떤 컴포넌트를 먼저 개선해야 할지 판단하기 어려움

### 대상 사용자
프론트엔드 아키텍처를 신경 쓰는 개발자들, 특히 중대형 React 프로젝트를 관리하는 팀 리더들과 코드 품질을 개선하고 싶은 개발자들

---

## 💡 해결 방안

### 핵심 아이디어
컴포넌트 아키텍처 분석기는 React 컴포넌트를 정적 분석하여 아키텍처 품질을 측정하고, 구체적인 개선점을 제안하는 도구입니다. AST 파싱을 통해 컴포넌트의 복잡도, 결합도, 재사용성 등을 분석하고, 시각적 대시보드로 결과를 제공합니다.

### 작동 원리
1. **프로젝트 스캔**: TypeScript 프로젝트를 분석하여 모든 React 컴포넌트를 찾고 AST를 파싱
2. **품질 메트릭 계산**: 각 컴포넌트의 복잡도, props 개수, 의존성, 재사용성 지수 등을 계산
3. **패턴 검증**: 설정된 아키텍처 패턴(Atomic Design, FSD 등)과의 일치도를 검사
4. **시각화 및 리포트**: 인터랙티브 대시보드로 결과를 표시하고 개선 제안을 제공

### 주요 기능
- **아키텍처 품질 스코어**: 각 컴포넌트와 전체 프로젝트의 품질을 0-100점으로 측정
- **의존성 그래프 시각화**: D3.js로 컴포넌트 간 의존 관계를 그래프로 표시
- **리팩토링 제안**: 구체적인 개선점과 코드 예시를 포함한 액션 아이템 제공
- **패턴 준수도 체크**: 선택한 아키텍처 패턴과의 일치도를 시각적으로 표시

### 기술 스택 선택 이유
- **ts-morph**: TypeScript AST 파싱의 표준 라이브러리로 컴포넌트 구조 분석에 최적화
- **D3.js**: 복잡한 의존성 그래프를 인터랙티브하게 시각화하는데 강력한 도구
- **React + TypeScript**: 타겟 사용자층과 같은 기술 스택으로 개발하여 친숙함 제공
- **Vite**: 빠른 개발 환경과 번들링으로 개발 생산성 향상

### 주요 기술적 과제
1. **AST 분석의 정확성**: React 컴포넌트의 다양한 작성 패턴을 모두 인식하고 분석하는 로직 구현
2. **대용량 프로젝트 성능**: 수백 개의 컴포넌트를 가진 프로젝트도 빠르게 분석할 수 있는 성능 최적화
