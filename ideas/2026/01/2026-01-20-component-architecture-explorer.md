# 컴포넌트 아키텍처 탐험가

> 리액트 컴포넌트를 메트로이드바니아 게임처럼 탐험하며 아키텍처를 직관적으로 이해하세요

## 🎯 문제점

### 현재 상황
대규모 React 프로젝트에서 컴포넌트 구조를 파악하는 것은 미로를 헤매는 것과 같습니다. 개발자들은 코드를 작성하는 시간보다 기존 컴포넌트 관계를 이해하는 데 더 많은 시간을 소모합니다. 기존의 정적 의존성 그래프는 탐험적 학습의 재미와 직관성을 제공하지 못합니다.

### 주요 문제점
- **인지 과부하**: 하나의 기능을 이해하기 위해 20개 이상의 파일을 왔다갔다해야 함
- **맥락 손실**: 파일을 찾는 동안 원래 목적을 잊어버리는 현상
- **진행도 추적 불가**: 어떤 컴포넌트를 탐험했고 어떤 것이 새로운지 파악하기 어려움
- **관계성 파악 어려움**: Props 전달과 컴포넌트 합성 패턴을 시각적으로 이해하기 힘듦

### 누가 이 문제를 겪나요?
- 새로운 팀에 합류한 프론트엔드 개발자
- 100개 이상의 컴포넌트를 가진 모노레포를 관리하는 개발자
- 몇 달 후 자신의 프로젝트에 돌아온 개발자
- 코드 리뷰 시 전체 맥락을 파악해야 하는 시니어 개발자

## 💡 해결책

### 핵심 컨셉
컴포넌트 아키텍처 탐험가는 React 컴포넌트를 3D 공간의 탐험 가능한 세계로 변환합니다. 각 컴포넌트는 방(room)이 되고, props와 imports는 문(door)이 되어 자연스러운 탐험을 유도합니다. 탐험할수록 맵이 채워지며, 컴포넌트 아키텍처에 대한 공간적 기억을 형성하게 됩니다. 정적 그래프와 달리, 이는 코드베이스 학습을 어드벤처 게임처럼 만들어줍니다.

### 작동 방식
1. **프로젝트 로드**: 리액트 프로젝트 루트를 선택하여 컴포넌트 구조 분석
2. **시작점 설정**: App.tsx나 index.tsx를 '스폰 포인트'로 설정하여 탐험 시작
3. **컴포넌트 탐험**: import문이나 props를 클릭하여 다른 컴포넌트 '방'으로 이동
4. **맵 생성**: 탐험한 영역은 맵에 표시되고, 미탐험 영역은 회색으로 표시
5. **파워업 획득**: 충분히 탐험하면 'jump to definition' 등의 단축키 해제

### 주요 기능
- **기억 맵**: 탐험한 컴포넌트의 시각적 맵 자동 생성, 미탐험 영역은 회색으로 표시
- **백트래킹**: 탐험 히스토리를 통해 이전 위치로 되돌아가기
- **파워업 시스템**: 탐험 진행도에 따라 새로운 네비게이션 능력 해제
- **컴포넌트 상태 시각화**: Props 흐름과 상태 관리를 3D 연결선으로 표현
- **미니맵**: 전체 아키텍처의 축소된 2D 뷰 제공

## 🏗️ 기술 아키텍처

### 시스템 개요
```
┌─────────────────┐
│ TypeScript      │ (컴포넌트 구조 분석)
│ AST Parser      │
└────────┬────────┘
         │
┌────────▼────────┐
│ Graph Model     │ (컴포넌트 관계 모델링)
│ Manager         │
└────────┬────────┘
         │
┌────────▼────────┐
│ Three.js        │ (3D 월드 렌더링)
│ Scene Manager   │
└────────┬────────┘
         │
┌────────▼────────┐
│ React UI        │ (인터페이스 & 컨트롤)
│ Components      │
└─────────────────┘
```

### 기술 스택 근거
- **Three.js**: WebGL을 직접 작성하지 않고도 3D 렌더링과 카메라 컨트롤을 제공. 공간 네비게이션에 최적화된 성숙한 생태계
- **TypeScript AST Parser**: 처음부터 파서를 구축하는 대신 기존 도구(ts-morph) 활용으로 AST 분석 가능
- **D3.js**: 복잡한 그래프 레이아웃 알고리즘 제공, Three.js와 연동하여 2D/3D 하이브리드 시각화
- **React**: 3D 월드와 전통적인 UI 컴포넌트 간의 일관된 상태 관리

### 주요 기술적 도전과제
1. **3D 레이아웃 알고리즘**: 컴포넌트를 공간적으로 의미있게 배치하는 방법? Force-directed 그래프나 계층적 트리 레이아웃 적용 고려
2. **대규모 코드베이스 성능**: 1000개 이상의 컴포넌트를 3D 객체로 처리. LOD(Level of Detail) 시스템이나 청크 로딩 필요
3. **실시간 상호작용**: 코드 변경 시 3D 월드의 실시간 업데이트, 웹소켓이나 파일 와처 연동

- 프로그레시브 디스클로저를 통한 복잡성 관리
- 메트로이드바니아 게임의 '능력 해제' 메커니즘을 개발 도구에 적용

### 학습 기회

- Force-directed 그래프 알고리즘
- WebGL 성능 최적화 패턴
- 게임 디자인 원리의 생산성 도구 적용
