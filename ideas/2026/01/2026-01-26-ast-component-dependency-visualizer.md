# AST 기반 컴포넌트 의존성 시각화 도구

> 코드를 수정하는 순간 실시간으로 업데이트되는 인터랙티브한 컴포넌트 관계도

## 🎯 문제 상황

### 현재 상황
React 모노레포에서 컴포넌트 하나를 수정할 때마다 "이 컴포넌트를 어디서 사용하고 있지?"라는 질문에 시달립니다. IDE의 "Find Usages" 기능은 텍스트 기반 검색에 불과하고, 전체적인 아키텍처 그림을 보여주지 못합니다. 개발자는 마치 미로 속에서 길을 찾듯 수십 개의 파일을 오가며 의존성을 파악해야 합니다.

### 주요 문제점
- **정신적 부담**: 하나의 컴포넌트 수정이 미치는 영향 범위를 파악하기 위해 20개 이상의 파일을 열어야 함
- **실시간 추적 불가**: 코드 변경 시 의존성 관계가 어떻게 바뀌는지 즉시 알 수 없음
- **아키텍처 가시성 부족**: 컴포넌트 간 순환 참조나 복잡한 의존성 체인을 직관적으로 파악하기 어려움

### 누가 이 문제를 겪나요?
100개 이상의 컴포넌트를 가진 React/TypeScript 모노레포에서 작업하는 프론트엔드 개발자들, 특히 기존 프로젝트에 새로 합류한 개발자들이나 레거시 코드를 리팩토링하는 팀원들이 가장 큰 고통을 받습니다.

## 💡 해결 방안

### 핵심 개념
AST 기반 컴포넌트 의존성 시각화 도구는 TypeScript 코드를 실시간으로 분석하여 컴포넌트 간의 의존성을 인터랙티브한 그래프로 시각화합니다. 파일을 저장하는 순간 그래프가 자동으로 업데이트되며, 개발자는 마우스 클릭만으로 특정 컴포넌트의 영향 범위를 탐색할 수 있습니다. 단순한 정적 분석을 넘어서서, 실제 import/export 관계와 props 전달 패턴을 분석하여 더 정확한 의존성 맵을 제공합니다.

### 작동 방식
1. **프로젝트 스캔**: 지정된 React 프로젝트의 모든 TypeScript 파일을 ts-morph로 파싱
2. **의존성 그래프 구축**: AST 분석을 통해 실제 import/export와 컴포넌트 사용 패턴 추출
3. **실시간 시각화**: D3.js 포스 다이렉티드 그래프로 컴포넌트 관계를 동적으로 렌더링
4. **인터랙티브 탐색**: 노드 클릭으로 해당 컴포넌트와 연결된 모든 관계 하이라이트

### 주요 기능
- **실시간 업데이트**: 파일 변경 감지 시 자동으로 그래프 재구성, 변경된 부분 애니메이션으로 표시
- **영향도 분석**: 특정 컴포넌트 선택 시 직접/간접 의존성을 색상으로 구분하여 표시
- **순환 참조 감지**: 문제가 될 수 있는 순환 의존성을 빨간색으로 강조 표시
- **필터링 시스템**: 파일 경로, 컴포넌트 타입, 의존성 깊이별로 그래프 필터링 가능

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────┐
│  File Watcher   │ (chokidar)
└────────┬────────┘
         │
┌────────▼────────┐
│  AST Parser     │ (ts-morph)
└────────┬────────┘
         │
┌────────▼────────┐
│ Dependency      │ (그래프 데이터 모델)
│ Graph Model     │
└────────┬────────┘
         │
┌────────▼────────┐
│ D3.js Renderer  │ (포스 다이렉티드 레이아웃)
└────────┬────────┘
         │
┌────────▼────────┐
│ React UI        │ (컨트롤 패널, 필터)
└─────────────────┘
```

### 기술 스택 선택 이유
- **ts-morph**: TypeScript 컴파일러 API 기반으로 정확한 AST 분석 가능, import/export 관계를 정밀하게 추출
- **D3.js**: 복잡한 그래프 데이터의 포스 다이렉티드 레이아웃과 부드러운 애니메이션 처리에 최적화
- **React**: 필터링 UI와 상태 관리를 위한 컴포넌트 기반 아키텍처
- **Vite**: 빠른 개발 서버와 HMR로 개발 경험 향상

### 주요 기술적 도전 과제
1. **대용량 프로젝트 성능**: 1000개 이상의 컴포넌트를 가진 프로젝트에서도 3초 이내에 그래프 렌더링 완료해야 함. 가상화와 청크 로딩으로 해결 예정
2. **실시간 업데이트 최적화**: 파일 변경 시 전체 그래프를 다시 그리지 않고 변경된 노드와 엣지만 업데이트하는 증분 렌더링 알고리즘 구현

## 🎨 이 프로젝트의 독특한 점

### 차별화 요소
기존의 의존성 분석 도구들은 대부분 정적인 그래프를 제공하거나 텍스트 기반 결과만 보여줍니다. 이 도구는 **코드 변경과 동시에 시각화가 업데이트되는 실시간성**과 **AST 기반의 정확한 분석**을 결합했습니다. 단순히 import 문을 찾는 것이 아니라 실제 컴포넌트 사용 패턴까지 분석하여 더 의미 있는 의존성 맵을 제공합니다.

### 혁신 포인트
- 실시간 코드 분석과 즉각적인 시각적 피드백의 결합

- 포스 다이렉티드 그래프를 통한 자연스러운 클러스터링과 의존성 패턴 발견
- 개발자 워크플로우에 자연스럽게 통합되는 실시간 업데이트

### 학습 기회
- AST 파싱과 코드 정적 분석 기법
- D3.js 포스 시뮬레이션과 대용량 데이터 시각화
- 실시간 파일 시스템 감시와 증분 업데이트 패턴
- 복잡한 그래프 데이터의 효율적인 상태 관리
