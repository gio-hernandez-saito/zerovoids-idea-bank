# 색상 공간 변환 라이브러리

> HSL, HCT, OKLCH 간 정확한 변환과 접근성을 고려한 색상 조작을 제공하는 TypeScript 라이브러리

## 🎯 문제점

### 현재 상황
웹 개발에서 색상을 다룰 때 HSL, RGB, HEX만으로는 한계가 있습니다. 최신 브라우저는 OKLCH, HCT 등 더 정확한 색상 공간을 지원하지만, 기존 도구들은 이를 제대로 활용하지 못하고 있습니다. 특히 접근성을 고려한 색상 조작은 매번 수동으로 계산해야 하는 상황입니다.

### 주요 문제점
- **부정확한 색상 조작**: HSL 기반 밝기 조정이 실제 인지 밝기와 다름
- **접근성 계산의 복잡성**: WCAG, APCA 대비율을 매번 수동으로 확인
- **색상 공간 간 변환 어려움**: OKLCH ↔ HSL 변환을 위한 신뢰할 수 있는 도구 부족

### 누가 이 문제를 겪나요?
디자인 시스템을 구축하는 프론트엔드 개발자, 접근성을 중시하는 UI 개발자, 색상 이론을 적용하려는 데이터 시각화 개발자

---

## 💡 해결책

### 핵심 개념
현대적인 색상 공간(OKLCH, HCT)과 기존 색상 공간(HSL, RGB) 간의 정확한 변환을 제공하는 TypeScript 라이브러리입니다. 단순한 변환뿐만 아니라 인지적으로 균일한 색상 조작과 접근성 검증 기능을 통합 제공합니다. npm 패키지로 배포되어 "설치하면 바로 동작"하는 것을 목표로 합니다.

### 동작 방식
1. **색상 입력**: 모든 주요 색상 형식을 파싱 (hex, hsl, oklch, hct 등)
2. **정확한 변환**: 색상 과학에 기반한 수학적 변환 알고리즘 적용
3. **접근성 검증**: WCAG/APCA 기준으로 대비율 자동 계산 및 개선 제안

### 주요 기능
- **통합 색상 API**: `Color.from('#ff0000').to('oklch')` 같은 직관적 인터페이스
- **인지 균일 조작**: OKLCH 기반으로 진짜 밝기/채도 조정
- **접근성 도우미**: `color.contrastWith(background).meets('WCAG-AA')` 자동 검증
