# 컴포넌트 아키텍처 분석기

> React 프로젝트의 설계 패턴과 아키텍처 품질을 시각적으로 분석하고 개선 방향을 제시하는 도구

## 🎯 문제 상황

### 현재 상황
React 프로젝트가 커질수록 컴포넌트 설계가 일관성을 잃고 복잡해집니다. 개발자들은 "이 컴포넌트가 너무 많은 일을 하고 있나?", "prop drilling이 심한가?", "컴포넌트 분리를 제대로 했나?"와 같은 의문을 갖지만, 이를 객관적으로 측정하고 시각화할 도구가 부족합니다.

### 주요 고민점
- **설계 품질 불투명**: 컴포넌트가 SRP(Single Responsibility Principle)을 지키는지 알기 어려움
- **의존성 복잡도**: Props drilling 깊이나 context 사용 패턴이 복잡한지 파악하기 어려움
- **리팩토링 우선순위**: 어떤 컴포넌트를 먼저 개선해야 할지 감이 안 옴
- **팀 코드 리뷰**: 아키텍처 관점에서 객관적인 피드백을 주기 어려움

### 누가 겪는 문제인가?
중급 이상의 React 개발자, 특히 컴포넌트 설계에 관심이 많고 Clean Architecture나 Design Pattern을 실무에 적용하려는 개발자들. 코드 리뷰에서 아키텍처 품질을 논의하고 싶은 팀들.

## 💡 해결책

### 핵심 아이디어
컴포넌트 아키텍처 분석기는 React 프로젝트의 AST를 분석해서 각 컴포넌트의 설계 품질을 측정하고, 이를 직관적인 시각화로 보여줍니다. 단순한 의존성 그래프가 아니라, Atomic Design, Container/Presenter Pattern, SRP 준수도 등 실제 설계 원칙 관점에서 분석합니다.

### 동작 방식
1. **프로젝트 스캔**: React 프로젝트 루트를 분석해서 모든 컴포넌트 추출
2. **패턴 분석**: 각 컴포넌트의 책임, 의존성, 복잡도, 재사용성을 측정
3. **시각화**: D3.js로 컴포넌트 간 관계와 품질 지표를 인터랙티브하게 표시
4. **개선 제안**: 문제가 있는 컴포넌트에 대해 구체적인 리팩토링 방향 제시

### 핵심 기능
- **아키텍처 스코어**: 각 컴포넌트의 SRP, 응집도, 결합도를 점수화
- **패턴 감지**: Atomic Design 레벨 자동 분류, HOC/Hooks 패턴 식별
- **의존성 시각화**: Props drilling 경로와 Context 사용 패턴을 3D 네트워크로 표시
- **리팩토링 가이드**: "이 컴포넌트는 3개로 분리 추천", "이 props는 context로 대체 추천" 등 구체적 조언

## 🏗️ 기술 아키텍처

### 시스템 구조

```
┌─────────────────┐
│  AST Parser     │ (TypeScript Compiler API)
└────────┬────────┘
         │
┌────────▼────────┐
│ Pattern Engine  │ (설계 패턴 분석)
└────────┬────────┘
         │
┌────────▼────────┐
│ Scoring System  │ (품질 지표 계산)
└────────┬────────┘
         │
┌────────▼────────┐
│ Visualization   │ (D3.js + Three.js)
└────────┬────────┘
         │
┌────────▼────────┐
│   Web UI        │ (React + TypeScript)
└─────────────────┘
```

### 기술 스택 선택 이유
- **TypeScript Compiler API**: React/JSX 파싱에 가장 정확하고 타입 정보까지 활용 가능
- **D3.js + Three.js**: 2D 네트워크 그래프와 3D 의존성 시각화를 모두 지원
- **React + TypeScript**: 익숙한 스택으로 빠른 개발, 자체 분석 대상이기도 함
- **Vite**: 빠른 개발 환경, HMR로 분석 결과 실시간 확인

### 주요 기술적 도전
1. **AST 분석 정확도**: JSX 패턴, Hooks 사용, Props 타입 등을 정확히 파싱
2. **성능 최적화**: 대형 프로젝트(1000+ 컴포넌트)에서도 실시간 분석 가능하도록 최적화
3. **패턴 인식 알고리즘**: 실제 개발자가 "좋은 설계"라고 느끼는 패턴을 수치화
