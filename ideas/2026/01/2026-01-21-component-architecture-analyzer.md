# 컴포넌트 아키텍처 분석기

> React 컴포넌트의 설계 패턴과 아키텍처 품질을 자동으로 분석하고 시각화하는 도구

## 🎯 문제 상황

### 현재 상황
프론트엔드 개발에서 컴포넌트 아키텍처는 프로젝트 성공의 핵심 요소입니다. 하지만 대부분의 개발자들은 컴포넌트를 "작동하게" 만드는 데 집중하고, 아키텍처 품질은 뒷전이 됩니다. 코드 리뷰에서도 "이게 좋은 구조인가?"라는 질문에 명확한 답을 하기 어렵습니다.

### 문제점
- **주관적 판단**: 컴포넌트 구조의 좋고 나쁨을 객관적으로 측정하기 어려움
- **숨겨진 복잡성**: Props drilling, 순환 의존성 등이 눈에 보이지 않음
- **패턴 인식 부족**: 어떤 디자인 패턴이 사용되고 있는지 파악하기 어려움
- **리팩토링 방향성 부재**: 어디서부터 개선해야 할지 모르겠음

### 누가 겪는 문제인가?
중급 이상의 React 개발자들, 특히 모노레포나 대규모 프로젝트를 관리하는 팀에서 컴포넌트 품질 관리에 고민이 많은 개발자들이 겪는 문제입니다.

## 💡 해결책

### 핵심 아이디어
컴포넌트 아키텍처 분석기는 React 프로젝트의 컴포넌트들을 자동으로 분석하여 아키텍처 패턴을 식별하고, 품질 지표를 측정하며, 개선점을 시각적으로 제시하는 도구입니다. AST 파싱을 통해 컴포넌트 구조를 파악하고, 설계 원칙 준수 여부를 체크합니다.

### 작동 방식
1. **프로젝트 스캔**: React 프로젝트 폴더를 분석하여 모든 컴포넌트 파일을 찾아냄
2. **AST 분석**: TypeScript/JavaScript 파서로 컴포넌트 구조, Props, Hook 사용 패턴을 추출
3. **패턴 매칭**: 컴포지션, HOC, Render Props, Compound Components 등 패턴 식별
4. **품질 측정**: 복잡도, 결합도, 응집도 등 객관적 지표 계산
5. **시각화**: D3.js로 컴포넌트 관계도와 품질 히트맵 생성

### 주요 기능
- **아키텍처 패턴 식별**: 사용된 디자인 패턴을 자동으로 인식하고 분류
- **복잡도 측정**: 컴포넌트별 순환 복잡도와 인지적 복잡도 계산
- **의존성 시각화**: Props drilling 경로와 Context 사용 패턴을 그래프로 표시
- **품질 점수**: 단일 책임 원칙, 개방-폐쇄 원칙 등 SOLID 원칙 준수도 점수화
- **리팩토링 제안**: 구체적인 개선 방안과 Before/After 예시 코드 제공

### 기술 스택 선택 이유

- **D3.js**: 복잡한 컴포넌트 관계도와 히트맵 시각화에 최적화된 라이브러리
- **React + Vite**: 빠른 개발 환경과 모던한 번들링으로 개발자 도구다운 UX 구현

### 주요 기술적 과제
1. **패턴 인식 정확도**: 다양한 React 패턴들을 높은 정확도로 식별하는 알고리즘 개발
2. **대용량 프로젝트 처리**: 수백 개 컴포넌트가 있는 프로젝트도 실시간으로 분석 가능한 성능 최적화
