# 컬러 공간 변환기

> HSL, OKLCH, HCT를 넘나드는 시각적 색상 변환 도구

## 🎯 문제 상황

### 현재 상황
디자이너와 개발자들은 여전히 HEX와 RGB에 의존하고 있지만, 현대적인 컬러 시스템은 HSL, OKLCH, HCT 같은 더 직관적인 색공간을 요구합니다. 특히 접근성을 고려한 디자인 시스템 구축 시 색상 대비를 정확히 계산하려면 지각적으로 균등한 색공간이 필요하지만, 기존 도구들은 이런 변환을 제대로 지원하지 않습니다.

### 주요 문제점
- **색공간 간 변환의 복잡함**: OKLCH에서 HSL로의 변환이 직관적이지 않음
- **접근성 검증의 어려움**: WCAG와 APCA 기준을 동시에 확인할 수 있는 도구 부족
- **디자인 시스템 일관성**: 브랜드 컬러를 다양한 색공간으로 표현할 때의 일관성 문제

### 누가 이 문제를 겪나요?
- 디자인 시스템을 구축하는 프론트엔드 개발자
- 접근성을 고려한 색상 팔레트를 만드는 UI/UX 디자이너
- 현대적인 CSS 색상 문법(OKLCH, HCT)을 도입하려는 개발팀

## 💡 해결책

### 핵심 컨셉
컬러 공간 변환기는 HSL, OKLCH, HCT, RGB, HEX 간의 실시간 변환을 시각적으로 보여주는 웹 도구입니다. 색상을 조정하면 모든 색공간의 값이 동시에 업데이트되며, 접근성 검증과 색상 조화 분석까지 한 번에 제공합니다. 단순한 변환 도구를 넘어서 색상 이론을 학습하고 실험할 수 있는 인터랙티브 플레이그라운드 역할을 합니다.

### 작동 방식
1. **색상 선택**: 컬러 피커, HEX 입력, 또는 색공간별 슬라이더로 색상 설정
2. **실시간 변환**: 선택한 색상이 모든 색공간에서 동시에 표현되며 값 자동 계산
3. **접근성 분석**: WCAG, APCA 기준으로 대비율 검증 및 권장사항 제시
4. **팔레트 생성**: 선택한 색상을 기반으로 조화로운 색상 조합 자동 생성

### 주요 기능
- **멀티 색공간 동기화**: 하나의 색상을 6가지 색공간에서 동시에 확인 가능
- **접근성 검증 대시보드**: WCAG 2.1, WCAG 3.0(APCA) 기준 실시간 검사
- **색상 조화 분석기**: 보색, 삼원색, 아날로그 색상 등 색상 이론 기반 팔레트 제안
- **CSS 코드 생성**: 선택한 색상의 CSS 변수 및 유틸리티 클래스 자동 생성

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────┐
│   색상 입력부    │ (컬러 피커, 슬라이더)
└────────┬────────┘
         │
┌────────▼────────┐
│   변환 엔진     │ (색공간 수학 연산)
└────────┬────────┘
         │
┌────────▼────────┐
│   시각화 층     │ (React 컴포넌트)
└────────┬────────┘
         │
┌────────▼────────┐
│   분석 모듈     │ (접근성, 조화 분석)
└─────────────────┘
```

### 기술 스택 근거
- **React + TypeScript**: 복잡한 색상 상태 관리와 실시간 업데이트를 위한 컴포넌트 기반 아키텍처. 타입 안전성으로 색공간 변환 오류 방지
- **CSS3 (OKLCH, HCT 지원)**: 최신 브라우저의 네이티브 색공간 지원 활용으로 정확한 색상 표현
- **Vite**: 빠른 개발 환경과 ES6 모듈 기반 최적화된 빌드

### 주요 기술적 도전
1. **색공간 변환 정확도**: OKLCH ↔ HSL 변환 시 색상 손실 최소화. CIE LAB을 중간 단계로 활용한 정밀 변환 알고리즘 구현
2. **실시간 성능 최적화**: 슬라이더 조작 시 6개 색공간의 동시 업데이트. requestAnimationFrame과 디바운싱으로 60fps 유지

## 📊 시장 분석 및 검증

### 유사 프로젝트
- **Coolors.co**: 색상 팔레트 생성 도구. 하지만 OKLCH 같은 현대적 색공간 미지원
- **Contrast Ratio Checker**: 접근성 검증에 특화. 색공간 변환 기능 부족
- **HSL Color Picker**: HSL 전용. 다른 색공간과의 비교 불가

### 왜 이런 틈새가 존재하나?
현대적인 색공간(OKLCH, HCT)의 브라우저 지원이 최근에야 안정화되었고, 디자인 시스템 트렌드와 접근성 요구사항이 동시에 부상한 시점이기 때문입니다. 기존 도구들은 레거시 색공간에 머물러 있어 이런 새로운 요구를 충족하지 못합니다.

### 성공 지표
- [ ] 6개 색공간에서 0.01 이하의 변환 오차율 달성
- [ ] 슬라이더 조작 시 16ms 이하의 응답 시간 유지
- [ ] 디자인 시스템 팀에서 실제 업무에 활용 피드백 확보
