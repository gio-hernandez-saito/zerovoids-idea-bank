# 색상 공간 변환 라이브러리

> HSL, HCT, OKLCH 간 완벽한 색상 변환을 제공하는 경량 TypeScript 라이브러리

## 🎯 문제 상황

### 현재 상황
현대 웹 디자인에서 색상 접근성은 필수가 되었지만, 다양한 색상 공간 간 변환은 여전히 복잡합니다. HSL은 직관적이지만 지각적으로 균등하지 않고, OKLCH는 최신 표준이지만 브라우저 지원이 제한적입니다. HCT(Material You)는 Google만의 독점 구현체가 대부분입니다.

### 주요 고충
- **색상 일관성 부족**: HSL의 명도가 색조에 따라 다르게 보임
- **접근성 도구 부족**: WCAG와 APCA 대비율 계산이 각각 따로 구현됨
- **변환 복잡성**: 색상 공간 간 정확한 변환 공식을 찾기 어려움
- **라이브러리 무거움**: 기존 솔루션들이 너무 크거나 불완전함

### 누가 이 문제를 겪나요?
디자인 시스템을 구축하는 프론트엔드 개발자, 접근성을 고려하는 UI/UX 디자이너, 색상 이론을 실제로 적용해야 하는 개발팀들이 주로 겪는 문제입니다.

## 💡 해결책

### 핵심 아이디어
색상 공간 변환 라이브러리는 HSL, HCT, OKLCH 간의 정확한 변환을 제공하는 경량 TypeScript 패키지입니다. 각 색상 공간의 장점을 활용하면서도 서로 간 완벽한 변환이 가능하게 합니다. 접근성 검사 기능과 색상 팔레트 생성 도구까지 포함하여 개발자가 색상을 다룰 때 필요한 모든 기능을 하나의 패키지로 제공합니다.

### 작동 방식
1. **색상 입력**: 어떤 형식(hex, hsl, oklch 등)이든 파싱
2. **내부 변환**: 정확한 수학 공식으로 색상 공간 간 변환
3. **출력 형식**: 원하는 색상 공간으로 결과 반환
4. **접근성 검증**: WCAG, APCA 기준으로 대비율 자동 계산

### 주요 기능
- **완벽한 변환**: HSL ↔ HCT ↔ OKLCH 간 무손실 변환
- **접근성 내장**: WCAG 2.1, WCAG 3.0(APCA) 대비율 계산
- **팔레트 생성**: 지각적으로 균등한 색상 팔레트 자동 생성
- **Tree-shakable**: 필요한 기능만 번들에 포함

### 유사 프로젝트

- **culori** (github.com/Evercoder/culori): 다양한 색상 공간 지원하지만 접근성 기능 부족
- **@material/material-color-utilities**: HCT만 지원, 다른 색상 공간과의 변환 불가

### 왜 이런 공백이 존재하는가?
HCT는 2022년에 도입된 새로운 색상 공간이고, OKLCH는 2023년에야 주요 브라우저에서 지원되기 시작했습니다. 접근성 표준도 WCAG 3.0으로 진화하면서 APCA라는 새로운 대비율 계산 방식이 추가되었습니다. 기존 라이브러리들이 이런 최신 표준을 따라가지 못하고 있는 상황입니다.
