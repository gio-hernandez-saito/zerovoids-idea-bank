# 야구 투구 시퀀스 스토리텔러

> 투구 데이터를 영화 같은 서사로 변환해서 전략과 심리전을 시각적으로 보여주는 도구

## 🎯 문제 상황

### 현재 상황
야구 투구 데이터는 엄청나게 풍부하지만, 대부분 건조한 통계표나 단순한 그래프로만 표현됩니다. 코치들은 복잡한 투구 전략을 선수들에게 설명할 때 "첫 번째 스트라이크를 잡고, 그 다음엔..." 같은 추상적인 설명에 의존합니다. 팬들도 투수와 타자 간의 심리전이 어떻게 전개되는지 직관적으로 이해하기 어려워합니다.

### 주요 문제점
- **맥락 없는 데이터**: 개별 투구 통계는 많지만 시퀀스의 스토리가 보이지 않음
- **설명의 한계**: 코치가 "상황에 따른 투구 선택"을 말로만 설명하기 어려움  
- **몰입감 부족**: 정적인 차트로는 투수-타자 대결의 긴장감이 전달되지 않음
- **패턴 인식 어려움**: 성공적인 투구 시퀀스의 공통점을 찾기 힘듦

### 누가 이런 문제를 겪나요?
야구 코치들 (전략 설명용), 야구 해설자들 (방송 자료용), 데이터 기반 야구 팬들, 그리고 선수 분석을 하는 프론트오피스 직원들입니다.

## 💡 해결 방안

### 핵심 컨셉
야구 투구 시퀀스를 영화의 플롯처럼 시각화합니다. 각 투구는 스토리의 한 장면이고, 볼카운트 변화는 긴장감의 고조입니다. D3.js로 애니메이션된 타임라인을 만들고, Three.js로 3D 스트라이크존에서 투구 궤적을 보여줍니다. 단순한 데이터 나열이 아니라, "이 투수가 어떻게 타자를 제압했는가"의 서사를 만들어냅니다.

### 작동 방식
1. **데이터 로드**: MLB Statcast 데이터나 KBO 데이터를 업로드
2. **시퀀스 선택**: 특정 타석이나 이닝의 투구 시퀀스를 선택
3. **스토리 생성**: AI가 투구 선택의 의도와 결과를 자연어로 해석
4. **시각화 재생**: 애니메이션으로 투구 하나하나가 어떻게 스토리를 만드는지 보여줌

### 핵심 기능
- **3D 스트라이크존**: Three.js로 구현된 실제 투구 궤적과 위치 시각화
- **서사 타임라인**: D3.js 애니메이션으로 볼카운트와 긴장감의 변화를 드라마틱하게 표현
- **전략 해석**: 각 투구 선택의 의도를 텍스트로 설명 ("바깥쪽 슬라이더로 스윙을 유도")
- **비교 모드**: 성공한 시퀀스와 실패한 시퀀스를 나란히 비교
- **하이라이트 추출**: 게임 내 가장 드라마틱한 순간들을 자동으로 찾아줌

## 🏗️ 기술 아키텍처

### 시스템 개요
```
┌─────────────────┐
│   Data Parser   │ (CSV/JSON → 구조화된 투구 데이터)
└────────┬────────┘
         │
┌────────▼────────┐
│ Story Generator │ (투구 시퀀스 → 서사 구조)
└────────┬────────┘
         │
┌────────▼────────┐
│  Visualization  │ (D3.js 타임라인 + Three.js 3D)
└────────┬────────┘
         │
┌────────▼────────┐
│   React UI      │ (컨트롤 패널 + 애니메이션 제어)
└─────────────────┘
```

### 기술 스택 근거
- **React + TypeScript**: 복잡한 UI 상태 관리와 타입 안정성
- **D3.js**: 투구 시퀀스의 시간축 애니메이션과 부드러운 전환 효과
- **Three.js**: 3D 스트라이크존과 투구 궤적의 현실적인 렌더링
- **Vite**: 빠른 개발 환경과 Three.js 에셋 번들링

### 주요 기술적 도전
1. **데이터와 애니메이션 동기화**: D3 타임라인과 Three.js 3D 렌더링의 프레임 동기화 처리
2. **성능 최적화**: 수백 개 투구 데이터의 실시간 애니메이션 렌더링
3. **서사 생성 로직**: 투구 데이터에서 의미있는 스토리 패턴을 추출하는 알고리즘

- 스포츠 데이터 분석과 도메인 지식
