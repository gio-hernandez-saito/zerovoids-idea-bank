# Git 커밋 히스토리 3D 시각화 도구

> Git 커밋 히스토리를 시간과 공간이 있는 3차원 세계로 탐험하세요

## 🎯 문제 상황

### 현재 상황
개발자들은 프로젝트의 진화 과정을 이해하기 위해 `git log`나 GitHub의 히스토리 뷰를 사용합니다. 하지만 이런 도구들은 시간순으로 나열된 평면적인 리스트만 보여줄 뿐, 브랜치 간의 복잡한 관계나 개발자별 기여도 패턴을 직관적으로 파악하기 어렵습니다.

### 주요 문제점
- **시각적 복잡성**: 복잡한 브랜치 구조에서 merge 지점과 분기점을 파악하기 어려움
- **시간 감각 부족**: 개발 속도의 변화나 활발했던 시기를 한눈에 보기 힘듦
- **개발자 기여도**: 누가 언제 어떤 영역에 집중했는지 패턴을 발견하기 어려움
- **코드 변화량**: 커밋별 실제 변경사항의 크기와 영향도를 시각적으로 비교 불가

### 누가 겪는 문제인가?
- 새로운 프로젝트에 합류한 개발자
- 프로젝트 매니저나 팀 리드
- 코드 리뷰 시 전체적인 맥락을 파악하려는 개발자
- 오픈소스 기여자들

## 💡 해결책

### 핵심 개념
Git 커밋 히스토리를 3차원 공간에서 시각화하는 웹 애플리케이션입니다. 각 커밋은 3D 공간의 한 점으로 표현되고, 시간축(Z축), 브랜치(X축), 변경량(Y축 높이)으로 배치됩니다. 개발자별로 다른 색상과 모양을 사용하여 누가 언제 어떤 작업을 했는지 한눈에 파악할 수 있습니다.

### 작동 방식
1. **저장소 분석**: Git 저장소를 입력받아 커밋 히스토리, 브랜치 정보, 파일 변경사항을 추출
2. **3D 좌표 계산**: 각 커밋을 시간, 브랜치, 변경량 기준으로 3차원 좌표에 배치
3. **인터랙티브 탐색**: 마우스로 시점을 회전하고 확대/축소하며 특정 커밋 클릭 시 상세 정보 표시
4. **시간 여행**: 타임라인 슬라이더로 프로젝트의 진화 과정을 애니메이션으로 재생

### 주요 기능
- **3D 커밋 맵**: 각 커밋을 3차원 공간의 구체로 표현, 크기는 변경된 라인 수에 비례
- **개발자별 컬러 코딩**: 각 개발자마다 고유한 색상으로 구분, 기여도 패턴 시각화
- **브랜치 플로우**: 브랜치 생성과 merge 지점을 곡선으로 연결하여 흐름 표현
- **시간 애니메이션**: 프로젝트의 시간 흐름에 따른 개발 활동 변화를 애니메이션으로 재생
- **필터링 시스템**: 특정 개발자, 날짜 범위, 파일 타입별로 필터링 가능

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────┐
│   Git Parser    │ (커밋 히스토리 분석)
└────────┬────────┘
         │
┌────────▼────────┐
│  Data Processor │ (3D 좌표 계산)
└────────┬────────┘
         │
┌────────▼────────┐
│ Three.js Render │ (3D 시각화)
└────────┬────────┘
         │
┌────────▼────────┐
│   React UI      │ (컨트롤 + 정보 패널)
└─────────────────┘
```

### 기술 스택 선택 이유
- **Three.js**: WebGL 기반 3D 렌더링으로 수천 개의 커밋도 부드럽게 처리 가능
- **React**: UI 상태 관리와 컴포넌트 구조화, 필터링 컨트롤 구현에 최적화
- **TypeScript**: Git 데이터 구조가 복잡하므로 타입 안전성 필수
- **Vite**: 빠른 개발 환경과 Three.js 번들링 최적화

### 주요 기술적 도전
1. **성능 최적화**: 수천 개 커밋을 3D로 렌더링할 때 60fps 유지. LOD(Level of Detail) 시스템과 인스턴싱 활용 필요
2. **3D 레이아웃 알고리즘**: 브랜치별로 의미있는 X축 배치와 merge 지점의 자연스러운 곡선 연결
3. **Git 데이터 파싱**: 복잡한 브랜치 구조와 merge 히스토리를 정확히 해석하는 로직

## 🎨 독창적인 점

### 새로운 접근법
기존의 Git 시각화 도구들은 대부분 2D 그래프나 트리 구조로 표현합니다. 하지만 이 도구는 **시간을 물리적 공간으로** 만들어 개발 히스토리를 **탐험 가능한 3차원 세계**로 변환합니다. 마치 프로젝트의 "시간 지도"를 걸어다니는 것 같은 경험을 제공합니다.

### 혁신 포인트
- 시간축을 3차원 공간의 깊이로 표현하여 **시간 여행하는 듯한 직관적 경험**
- 개발자별 활동 패턴을 **우주의 별자리처럼** 시각화
- 브랜치 merge를 **3차원 곡선 흐름**으로 표현하여 코드의 합류 지점을 자연스럽게 시각화
- Git log의 텍스트 정보를 **공간적 메타포**로 변환

### 학습 기회
- Three.js 고급 기능: 인스턴싱, 셰이더 최적화, LOD 시스템
- 3D 수학: 벡터 계산, 카메라 컨트롤, 공간 변환
- 알고리즘 설계: 그래프 레이아웃, 시간 기반 애니메이션
- Git 내부 구조 깊이 이해

## 📊 시장 분석 및 검증

### 유사 프로젝트
- **GitKraken** (gitkraken.com): 2D 그래프 기반 Git GUI. 이 프로젝트는 3D 공간 활용으로 더 많은 정보를 직관적으로 표현
- **Gource** (gource.io): 파일 변화를 3D로 보여주지만 커밋 히스토리 자체는 시각화하지 않음
- **GitHub Network Graph**: 브랜치 구조만 표현, 시간과 개발자 패턴 정보 부족

### 왜 이런 도구가 없었나?
기존 도구들은 "정보 전달"에 초점을 맞췄지만, 이 프로젝트는 "탐험과 발견"에 중점을 둡니다. WebGL 성능이 충분해진 지금에서야 브라우저에서 부드러운 3D 시각화가 가능해졌습니다. 또한 Git 사용이 복잡해질수록 단순한 2D 표현의 한계가 명확해지고 있습니다.
