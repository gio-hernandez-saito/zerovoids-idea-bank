# 실시간 컴포넌트 성능 히트맵

> React 앱의 각 컴포넌트 성능을 실시간으로 시각화하여 병목지점을 한눈에 파악하세요

## 🎯 문제 상황

### 현재 상황
React 앱의 성능 문제를 찾을 때 개발자들은 React DevTools Profiler를 사용하지만, 수백 개의 컴포넌트 중 어디가 문제인지 찾기가 어렵습니다. 프로파일링 결과는 복잡한 플레임 그래프나 리스트 형태로만 제공되어 직관적이지 않습니다.

### 문제점들
- **인지 과부하**: 수백 개 컴포넌트의 성능 데이터를 텍스트/그래프로만 확인
- **실시간성 부족**: 프로파일링을 시작하고 멈춰야만 결과 확인 가능
- **공간적 맥락 부족**: UI 상에서 어떤 컴포넌트가 느린지 바로 연결되지 않음
- **패턴 파악 어려움**: 시간에 따른 성능 변화 추이를 보기 힘듦

### 누가 겪는 문제인가?
대규모 React 애플리케이션을 개발하는 프론트엔드 개발자들, 특히 성능 최적화를 담당하는 시니어 개발자들과 복잡한 상태 관리가 있는 대시보드나 데이터 시각화 앱을 만드는 팀들

## 💡 해결책

### 핵심 컨셉
실시간 컴포넌트 성능 히트맵은 React 앱의 실제 UI 위에 성능 데이터를 컬러 오버레이로 표시합니다. 각 컴포넌트의 렌더링 시간에 따라 초록색(빠름)에서 빨간색(느림)까지 히트맵으로 시각화하여, 개발자가 문제 영역을 즉시 식별할 수 있게 합니다. React DevTools API와 연동하여 실시간으로 데이터를 수집하고 업데이트합니다.

### 동작 방식
1. **확장 프로그램 활성화**: Chrome DevTools에서 히트맵 모드 활성화
2. **실시간 데이터 수집**: React DevTools API를 통해 각 컴포넌트의 렌더링 메트릭 수집
3. **시각적 오버레이**: 브라우저 화면 위에 투명한 컬러 오버레이로 성능 상태 표시
4. **상호작용**: 히트맵 영역 클릭 시 해당 컴포넌트의 상세 성능 정보 표시

### 주요 기능
- **실시간 히트맵**: UI 컴포넌트 위에 성능 상태를 색상으로 표시 (빨강=느림, 초록=빠름)
- **성능 임계값 설정**: 프로젝트별로 "느림" 기준을 커스터마이징
- **시간대별 히스토리**: 특정 컴포넌트의 성능 변화를 시간축으로 시각화
- **문제 컴포넌트 자동 하이라이트**: 설정한 임계값을 초과하는 컴포넌트를 자동으로 강조표시

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────┐
│ Chrome Extension│ (UI 오버레이)
└────────┬────────┘
         │
┌────────▼────────┐
│ React DevTools  │ (성능 데이터 수집)
│     Bridge      │
└────────┬────────┘
         │
┌────────▼────────┐
│ Performance     │ (데이터 처리 & 분석)
│   Analyzer      │
└────────┬────────┘
         │
┌────────▼────────┐
│ D3.js Heatmap   │ (색상 매핑 & 시각화)
│   Renderer      │
└─────────────────┘
```

### 기술 스택 선택 이유
- **Chrome Extension API**: 브라우저 페이지 위에 오버레이 렌더링이 가능하고 DevTools와 통신 지원
- **React DevTools API**: React 컴포넌트의 실제 성능 메트릭에 접근할 수 있는 유일한 공식 방법
- **D3.js**: 색상 스케일링과 히트맵 시각화에 최적화된 라이브러리
- **TypeScript**: 복잡한 성능 데이터 타입과 Chrome API 타입 안정성 보장

### 주요 기술적 도전과제
1. **실시간 데이터 동기화**: React DevTools에서 오는 성능 데이터와 브라우저 DOM 요소를 정확히 매핑하는 문제. Fiber 노드와 실제 DOM 요소 간의 관계를 추적해야 함
2. **성능 오버헤드 최소화**: 성능 측정 도구 자체가 성능에 영향을 주면 안 됨. 렌더링 루프 최적화와 효율적인 데이터 구조 설계 필요

## 📊 시장 및 검증

### 유사 프로젝트
- **React DevTools Profiler**: 플레임 그래프와 리스트 뷰 제공. 이 도구는 공간적 맥락과 실시간성을 추가

### 왜 이런 공백이 존재하는가?
React DevTools API가 상대적으로 최근에 안정화되었고, Chrome Extension에서 페이지 오버레이를 구현하는 것은 기술적으로 복잡합니다. 또한 성능 도구 개발자들이 전통적인 차트 방식에 익숙해져 있어 공간적 시각화 접근법을 시도하지 않았습니다.
