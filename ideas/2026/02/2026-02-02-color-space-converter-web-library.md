# 색상 공간 변환기 웹 라이브러리

> 모든 색상 공간을 자유롭게 변환하는 타입 안전한 웹 라이브러리

## 🎯 문제점

### 현재 상황
웹 개발에서 색상을 다룰 때 다양한 색상 공간(HSL, RGB, OKLCH, HCT 등)을 오가며 작업해야 한다. 특히 접근성을 고려한 색상 선택이나 디자인 시스템 구축 시 정확한 색상 변환이 필수적이다. 하지만 현재 라이브러리들은 제한적이거나 사용법이 복잡하다.

### 주요 문제점
- **불완전한 변환**: 대부분 라이브러리가 RGB/HSL만 지원, OKLCH나 HCT 변환 누락
- **정확도 부족**: 색상 변환 시 감마 보정이나 색역 매핑 고려하지 않음
- **타입 안전성 부족**: JavaScript 환경에서 색상 값 실수로 인한 런타임 에러
- **접근성 도구 부재**: WCAG나 APCA 기준 색상 대비 계산 기능 부족

### 누가 겪는 문제인가?
UI/UX 개발자, 디자인 시스템 구축자, 접근성을 중요시하는 프론트엔드 개발자들이 색상 작업 시 매번 겪는 문제다.

---

## 💡 해결책

### 핵심 컨셉
모든 주요 색상 공간(RGB, HSL, OKLCH, HCT, LAB 등)을 완벽하게 지원하는 타입 안전한 웹 라이브러리다. 단순한 변환을 넘어 접근성 계산, 색상 팔레트 생성, 색맹 시뮬레이션까지 포함한다. npm 패키지로 배포하여 "설치하면 바로 작동"하는 라이브러리를 목표로 한다.

### 작동 방식
1. **색상 입력**: 어떤 형식(hex, rgb, hsl, oklch 등)이든 파싱
2. **내부 변환**: 표준 색상 공간으로 정규화 후 목표 공간으로 변환
3. **결과 출력**: 타입 안전한 객체로 반환, 체이닝 가능

### 주요 기능
- **완전한 색상 공간 지원**: RGB, HSL, OKLCH, HCT, LAB, XYZ 등 모든 주요 공간
- **접근성 도구**: WCAG 2.1/3.0, APCA 대비 계산 내장
- **타입 안전성**: TypeScript로 색상 값 검증 및 자동완성 지원
- **체이닝 API**: `color('#ff0000').to('oklch').darken(0.2).toString()`

2. **성능 최적화**: 실시간 색상 변환이 가능하도록 계산 로직 최적화 필요
