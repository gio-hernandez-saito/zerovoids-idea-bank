# React 컴포넌트 아키텍처 진단기

> 복잡한 React 프로젝트의 컴포넌트 아키텍처를 건강 검진하듯 진단하고 개선점을 제안하는 도구

## 🎯 문제 상황

### 현재 상황
대규모 React 프로젝트에서는 시간이 지날수록 컴포넌트들이 서로 복잡하게 얽혀간다. 개발자들은 컴포넌트를 수정할 때 어디까지 영향이 미칠지 예측하기 어렵고, 리팩토링을 시도하다가 예상치 못한 부작용을 경험한다. 특히 모노레포 환경에서는 여러 패키지 간 의존성까지 고려해야 해서 상황이 더욱 복잡해진다.

### 고충 지점
- **숨겨진 의존성**: prop drilling이나 context 사용으로 인한 보이지 않는 연결고리들
- **순환 의존성**: 컴포넌트들이 서로를 참조하면서 생기는 복잡한 관계망
- **아키텍처 부채**: 일관성 없는 패턴 사용으로 인한 유지보수 어려움
- **영향 범위 파악 불가**: 한 컴포넌트 변경이 어디까지 영향을 미칠지 모름

### 누가 겪는 문제인가?
100개 이상의 컴포넌트를 가진 React 프로젝트를 관리하는 프론트엔드 개발자들, 특히 모노레포 환경에서 여러 앱을 동시에 관리하는 팀과 새로 합류한 개발자들이 코드베이스 구조를 파악해야 하는 상황

## 💡 해결책

### 핵심 개념
React 컴포넌트 아키텍처 진단기는 의료진이 환자를 진단하듯 React 프로젝트의 건강 상태를 체크한다. AST 파싱을 통해 컴포넌트 간 관계를 분석하고, 아키텍처 안티패턴을 탐지하며, 개선 방향을 시각적으로 제안한다. 단순한 의존성 그래프를 넘어서 실제 아키텍처 품질을 측정하고 구체적인 리팩토링 가이드를 제공한다.

### 작동 방식
1. **프로젝트 스캔**: React 프로젝트 루트를 분석하여 모든 컴포넌트와 hook을 파싱
2. **관계 분석**: props, context, custom hooks를 통한 의존성 관계를 매핑
3. **패턴 탐지**: 알려진 아키텍처 패턴과 안티패턴을 식별
4. **건강도 진단**: 복잡도, 결합도, 응집도 등의 메트릭으로 점수화
5. **개선안 제시**: 구체적인 리팩토링 방향과 우선순위를 시각화

### 핵심 기능
- **아키텍처 건강도 스코어**: 전체 프로젝트의 아키텍처 품질을 0-100점으로 측정
- **핫스팟 탐지**: 가장 문제가 되는 컴포넌트들을 빨간색으로 하이라이트
- **패턴 매칭**: Atomic Design, Container/Presentational 등 적용된 패턴 분석
- **리팩토링 우선순위**: 개선 효과가 큰 순서대로 작업 목록 제공

```
┌─────────────────┐

         │
┌────────▼────────┐
│  Web Dashboard  │ (React UI)
└─────────────────┘
```

### 기술 스택 선택 이유
- **TypeScript AST Parser**: 기존 도구 활용으로 파싱 구현 시간 단축, 타입 정보까지 분석 가능
- **D3.js**: 복잡한 컴포넌트 관계도를 네트워크 그래프로 시각화하는데 최적화
- **React**: 분석 대상과 동일한 기술로 구현하여 도메인 이해도 활용

### 주요 기술 과제
1. **대용량 프로젝트 처리**: 1000개+ 컴포넌트를 가진 프로젝트에서도 실시간 분석이 가능하도록 청킹과 워커 활용
2. **패턴 인식 알고리즘**: 휴리스틱 기반으로 다양한 아키텍처 패턴을 자동 탐지하는 규칙 엔진 설계

- 시간에 따른 아키텍처 변화 추적 가능

### 학습 기회

- 복잡한 네트워크 데이터 시각화
- 아키텍처 메트릭과 품질 측정 방법론
- 대용량 데이터 처리 최적화

### 유사 프로젝트
- **Madge** (github.com/pahen/madge): 일반적인 모듈 의존성 분석. 이 도구는 React 전용 패턴과 아키텍처 품질 측정에 특화
- **Dependency Cruiser**: 의존성 규칙 검증. 이 도구는 진단과 개선 가이드 제공에 초점

### 이런 갭이 존재하는 이유
React 생태계는 빠르게 변화하고 있고, hooks와 context API 같은 새로운 패턴들이 만드는 의존성은 기존 도구로는 제대로 분석하기 어렵다. 또한 "어떤 문제가 있는지"는 알 수 있어도 "어떻게 해결할지"까지 제시하는 도구는 부족하다.
