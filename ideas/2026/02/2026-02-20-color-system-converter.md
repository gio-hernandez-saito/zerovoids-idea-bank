# 컬러 시스템 변환기

> 디자인 시스템 컬러를 HSL, HCT, OKLCH 간 완벽하게 변환하고 실시간 접근성 검증까지

## 🎯 문제 상황

### 현재 상황
최신 디자인 시스템들이 OKLCH나 HCT 같은 새로운 컬러 스페이스로 이전하고 있지만, 기존 HSL 기반 시스템을 변환하는 과정이 복잡합니다. 각 컬러 스페이스마다 다른 도구가 필요하고, 접근성 검증은 별도로 해야 합니다.

### 핵심 문제점
- **컬러 스페이스 파편화**: HSL에서 OKLCH로 변환할 때 값이 맞지 않아 디자인이 깨짐
- **접근성 검증의 번거로움**: WCAG와 APCA를 각각 다른 도구로 확인해야 함
- **일괄 변환의 어려움**: 디자인 토큰 파일 전체를 한 번에 변환할 방법이 없음

### 누가 겪는 문제인가?
디자인 시스템을 관리하는 프론트엔드 개발자와 디자이너, 특히 최신 컬러 스페이스로 마이그레이션을 고민하는 팀들이 겪는 문제입니다.

## 💡 해결책

### 핵심 개념
모든 주요 컬러 스페이스(HSL, RGB, HCT, OKLCH)를 실시간으로 변환하면서 동시에 WCAG 2.1/3.0과 APCA 접근성 기준을 검증하는 통합 도구입니다. 단일 컬러뿐만 아니라 디자인 토큰 파일 전체를 일괄 변환하고 접근성 리포트를 생성합니다.

### 작동 방식
1. **컬러 입력**: 어떤 형식(hex, hsl, oklch 등)이든 붙여넣기
2. **실시간 변환**: 모든 컬러 스페이스로 동시 변환 및 미리보기
3. **접근성 검증**: 선택한 배경색과의 대비율을 WCAG/APCA로 자동 계산
4. **일괄 처리**: JSON/CSS 파일 업로드하여 전체 팔레트 변환

### 주요 기능
- **정확한 변환**: CIE Lab을 거친 과학적으로 정확한 컬러 스페이스 변환
- **실시간 미리보기**: 변환된 컬러가 실제 UI에서 어떻게 보일지 즉시 확인
- **접근성 대시보드**: WCAG 2.1, WCAG 3.0, APCA 기준 모두 한 화면에서 검증
- **파일 변환**: design-tokens.json이나 CSS 변수를 드래그앤드롭으로 일괄 변환
