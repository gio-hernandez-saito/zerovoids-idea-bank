# 웹 페이지 색 접근성 실시간 검증기

> 웹페이지를 보는 순간 색상 접근성 문제를 실시간으로 감지하고 개선안을 제시하는 브라우저 확장프로그램

## 🎯 문제 상황

### 현재 상황
웹 개발자들은 색상 접근성을 체크하기 위해 여러 도구를 왔다갔다 하며 수동으로 검증해야 합니다. WCAG 가이드라인을 만족하는지 확인하려면 색상을 추출하고, 대비율을 계산하고, 각각의 조합을 일일이 테스트해야 하는 번거로운 과정을 거쳐야 합니다.

### 문제점들
- **분절된 워크플로우**: 페이지 보기 → 개발자 도구 열기 → 색상 추출 → 외부 툴에서 검증
- **놓치기 쉬운 조합들**: 동적으로 생성되는 색상이나 상태별 색상 변화를 놓치기 쉬움
- **개선안 부재**: 문제는 찾아주지만 구체적인 해결책은 제시하지 않음

### 누가 이 문제를 겪나요?
접근성을 고려하는 프론트엔드 개발자, UI/UX 디자이너, 웹 접근성 감사를 담당하는 전문가들. 특히 정부기관이나 대기업에서 웹 접근성 준수가 필수인 프로젝트를 진행하는 팀들.

## 💡 해결방안

### 핵심 개념
브라우저 확장프로그램으로 웹페이지의 모든 색상 조합을 실시간으로 분석하여 접근성 문제를 즉시 표시하고, OKLCH 색공간을 활용한 정확한 개선안을 제시합니다. 단순히 문제를 찾는 것을 넘어서, 어떻게 고치면 되는지까지 구체적으로 알려주는 도구입니다.

### 작동 방식
1. **페이지 스캔**: 현재 페이지의 모든 텍스트-배경 색상 조합을 자동 감지
2. **실시간 분석**: OKLCH 색공간에서 정확한 대비율 계산 및 WCAG 준수 여부 판단
3. **시각적 표시**: 문제가 있는 요소들을 페이지에서 직접 하이라이트
4. **개선안 제시**: 최소한의 색상 변경으로 접근성을 만족하는 대안 색상 추천

### 주요 기능
- **실시간 하이라이팅**: 접근성 문제가 있는 텍스트를 페이지에서 즉시 표시
- **스마트 색상 제안**: OKLCH 기반으로 브랜드 아이덴티티를 유지하면서 접근성을 개선하는 색상 추천
- **시뮬레이션 모드**: 색맹/색약 사용자 관점에서 페이지가 어떻게 보이는지 실시간 미리보기

## 🏗️ 기술 아키텍처

### 시스템 개요
```
┌─────────────────┐
│  Content Script │ (페이지 색상 추출)
└────────┬────────┘
         │
┌────────▼────────┐
│ Color Analyzer  │ (OKLCH 계산)
└────────┬────────┘
         │
┌────────▼────────┐
│ WCAG Validator  │ (접근성 검증)
└────────┬────────┘
         │
┌────────▼────────┐
│ UI Overlay      │ (결과 표시)
└─────────────────┘
```

### 기술 스택 선택 이유
- **Browser Extension APIs**: 모든 웹페이지에서 작동하고 DOM에 직접 접근 가능
- **OKLCH 색공간**: sRGB보다 인간의 시각 인지에 더 정확한 색상 계산
- **Canvas API**: 색상 추출과 시각적 시뮬레이션을 위한 픽셀 단위 조작
- **TypeScript**: 색상 계산의 정확성과 API 타입 안전성 보장

### 주요 기술적 도전과제
1. **성능 최적화**: 대용량 페이지에서도 빠른 색상 분석을 위한 효율적인 DOM 순회 알고리즘 필요
2. **동적 콘텐츠 감지**: SPA의 상태 변화나 호버/포커스 상태의 색상 변화까지 추적하는 방법

## 🎨 무엇이 흥미로운가?

### 독특한 접근방식
기존 접근성 도구들이 정적 분석에 머무르는 반면, 이 도구는 실제 사용자 경험 맥락에서 실시간으로 문제를 발견하고 해결책까지 제시합니다. 개발자가 페이지를 보면서 즉시 '아, 이 색상이 문제구나. 이렇게 바꾸면 되겠네!'라는 인사이트를 얻을 수 있습니다.

### 혁신 포인트
- OKLCH 색공간을 웹 접근성 도구에 적용한 선도적 사례
- 문제 발견과 해결책 제시를 하나의 워크플로우로 통합
- 브랜드 아이덴티티를 유지하면서 접근성을 개선하는 지능적 색상 추천

### 학습 기회
- OKLCH와 다른 색공간 간의 정확한 변환 알고리즘
- 브라우저 확장프로그램의 Content Script와 Background Script 아키텍처
- WCAG 가이드라인의 실제 구현과 색상 대비 계산법

## 📊 시장 현황 및 검증

### 유사 프로젝트들
- **WAVE** ([wave.webaim.org](https://wave.webaim.org)): 전반적인 접근성 검사 도구이지만 색상 특화 기능은 제한적
- **Colour Contrast Analyser**: 수동으로 색상을 입력해야 하는 데스크톱 앱
- **axe DevTools**: 포괄적이지만 실시간 색상 개선안 제시 기능 없음

### 왜 이런 틈새가 존재하는가?
접근성 도구들은 '무엇이 문제인지'는 잘 찾아주지만 '어떻게 고쳐야 하는지'까지는 알려주지 않습니다. 특히 색상 이론에 기반한 정확한 개선안 제시는 복잡한 색공간 계산이 필요해서 대부분의 도구가 회피하는 영역입니다.

### 성공 지표
- [ ] Chrome 웹스토어에서 주간 1000+ 다운로드
- [ ] 접근성 전문가 커뮤니티에서 언급 및 추천
- [ ] 실제 웹사이트 접근성 개선 사례 5개 이상 문서화
