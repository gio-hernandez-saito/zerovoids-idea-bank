# 컴포넌트 아키텍처 시각화 도구

> React 프로젝트의 컴포넌트 구조와 의존성을 인터랙티브한 네트워크 그래프로 탐험하세요

## 🎯 문제 상황

### 현재 상황
React 프로젝트가 커질수록 컴포넌트 간의 관계를 파악하기 어려워집니다. 개발자들은 props drilling이 어디서 발생하는지, 어떤 컴포넌트가 과도하게 재사용되고 있는지, 순환 의존성이 있는지를 알기 위해 수십 개의 파일을 직접 열어봐야 합니다.

### 고충 포인트
- **정신적 지도 부재**: 100개 이상의 컴포넌트가 어떻게 연결되어 있는지 머릿속으로 그리기 어려움
- **리팩토링 두려움**: 한 컴포넌트를 수정할 때 어디에 영향을 미칠지 예측하기 어려움
- **중복 컴포넌트 발견 불가**: 비슷한 기능의 컴포넌트가 여러 곳에 흩어져 있어도 모름
- **성능 핫스팟 파악 어려움**: 어떤 컴포넌트가 자주 리렌더링되는지 시각적으로 확인할 수 없음

### 누가 이런 문제를 겪나요?
대형 React 프로젝트를 유지보수하는 프론트엔드 개발자들, 특히 모노레포 환경에서 여러 팀이 공유하는 컴포넌트 라이브러리를 관리하는 개발자들이 가장 심각하게 경험합니다.

## 💡 해결책

### 핵심 컨셉
컴포넌트 아키텍처 시각화 도구는 React 프로젝트의 모든 컴포넌트를 인터랙티브한 네트워크 그래프로 표현합니다. 각 노드는 컴포넌트를, 연결선은 props 전달이나 import 관계를 나타냅니다. D3.js의 force simulation을 활용해 관련성이 높은 컴포넌트들이 자연스럽게 클러스터링되도록 하며, 사용자는 실시간으로 구조를 탐험하고 분석할 수 있습니다.

### 작동 방식
1. **프로젝트 스캔**: TypeScript AST 파서를 사용해 React 프로젝트의 모든 컴포넌트와 관계를 추출
2. **그래프 생성**: 컴포넌트를 노드로, 의존성을 엣지로 하는 그래프 데이터 구조 생성
3. **인터랙티브 시각화**: D3.js force simulation으로 동적인 네트워크 그래프 렌더링
4. **탐험적 분석**: 노드 클릭, 필터링, 검색을 통한 실시간 구조 분석

### 주요 기능
- **적응형 레이아웃**: Force-directed 알고리즘으로 관련 컴포넌트들이 자동으로 그룹화
- **다층 필터링**: 컴포넌트 타입, 의존성 깊이, 사용 빈도별로 선택적 표시
- **핫스팟 분석**: Props가 많이 전달되는 경로나 과도하게 import되는 컴포넌트 하이라이트
- **아키텍처 인사이트**: 순환 의존성, 고아 컴포넌트, 거대 컴포넌트 자동 탐지

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────┐
│ AST Parser      │ (TypeScript 코드 분석)
│ (ts-morph)      │
└────────┬────────┘
         │
┌────────▼────────┐
│ Graph Builder   │ (컴포넌트 관계 모델링)
└────────┬────────┘
         │
┌────────▼────────┐
│ D3.js Engine    │ (Force simulation)
└────────┬────────┘
         │
┌────────▼────────┐
│ React UI        │ (인터랙션 & 컨트롤)
└─────────────────┘
```

### 기술 스택 선택 근거
- **ts-morph**: TypeScript AST 파싱에 최적화되어 있고, React 컴포넌트의 props, imports를 정확히 추출 가능
- **D3.js**: Force simulation과 복잡한 그래프 레이아웃 알고리즘을 제공하며, 대용량 노드 처리 성능 우수
- **React + TypeScript**: 개발자 도구 특성상 타입 안전성이 중요하고, 컴포넌트 기반 UI 구조가 적합
- **Vite**: 빠른 개발 환경과 효율적인 번들링으로 개발자 경험 최적화

### 주요 기술적 도전과제
1. **대용량 그래프 성능**: 500+ 컴포넌트를 부드럽게 렌더링하기 위한 가상화와 LOD(Level of Detail) 구현 필요
2. **동적 레이아웃 최적화**: Force simulation 파라미터 튜닝으로 의미 있는 클러스터링과 가독성 사이의 균형점 찾기

### 혁신 포인트
- 물리학 기반 레이아웃으로 직관적인 컴포넌트 관계 표현

- Props flow와 component hierarchy를 동시에 시각화
- 정적 분석과 동적 시각화의 결합

### 학습 기회
- D3.js의 고급 force simulation과 그래프 레이아웃 알고리즘
- TypeScript AST 조작과 코드 정적 분석 기법
- 대용량 데이터 시각화에서의 성능 최적화 패턴
- React와 D3.js의 효율적인 통합 방법
