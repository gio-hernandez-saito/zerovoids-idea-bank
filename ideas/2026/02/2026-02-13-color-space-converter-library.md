# 컬러 스페이스 변환 라이브러리

> TypeScript로 작성된 현대적 컬러 스페이스 변환 라이브러리 - 접근성과 퍼셉션을 고려한 타입 안전한 API

## 🎯 문제 상황

### 현재 상황
현재 JavaScript 생태계의 컬러 라이브러리들은 대부분 오래된 RGB/HSL 중심으로 설계되어 있습니다. OKLCH, HCT같은 현대적 컬러 스페이스에 대한 지원이 부족하고, 접근성 고려사항(WCAG, APCA)이 후순위로 밀려납니다. 대부분의 라이브러리가 JavaScript로 작성되어 TypeScript 환경에서 타입 안전성이 부족합니다.

### 주요 문제점
- **접근성 고려 부족**: 색상 대비 계산이 오래된 WCAG 2.1 기준에만 의존
- **현대 컬러 스페이스 부재**: OKLCH, HCT 등 인지적으로 균등한 컬러 스페이스 미지원
- **타입 안전성 결여**: 런타임에서야 발견되는 잘못된 컬러 값들
- **성능 최적화 부족**: 브라우저별 네이티브 지원 활용 못함

### 누가 이 문제를 겪나요?
디자인 시스템 개발자, 접근성을 중요시하는 프론트엔드 개발자, 데이터 시각화 작업을 하는 개발자들이 이 문제를 직접 경험합니다.

## 💡 해결책

### 핵심 컨셉
접근성과 인지 과학을 기반으로 한 현대적 컬러 스페이스 변환 라이브러리입니다. TypeScript 퍼스트로 설계하여 컴파일 타임에 컬러 값의 유효성을 보장하고, OKLCH와 HCT 같은 최신 컬러 스페이스를 네이티브 지원합니다. APCA(Accessible Perceptual Contrast Algorithm) 같은 차세대 접근성 기준도 포함합니다.

### 작동 방식
1. **타입 안전한 컬러 생성**: `Color.oklch(70, 0.15, 180)` 같은 명시적 API
2. **스마트 변환**: 브라우저별 최적 경로로 컬러 스페이스 변환 수행
3. **접근성 자동 검증**: 생성된 컬러 조합의 대비율과 접근성 자동 체크

### 핵심 기능
- **현대적 컬러 스페이스 지원**: OKLCH, HCT, P3, Rec2020 네이티브 지원
- **강력한 TypeScript 타입**: 컬러 스페이스별 별도 타입으로 잘못된 값 방지
- **접근성 우선 API**: APCA, WCAG 3.0 draft 기준 포함
- **성능 최적화**: 브라우저 네이티브 CSS Color 4 활용
