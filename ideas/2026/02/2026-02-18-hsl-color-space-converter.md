# HSL 색상 공간 변환기

> HSL, HCT, OKLCH 간 실시간 변환과 접근성 점수를 한눈에 보는 도구

## 🎯 문제 상황

### 현재 상황
디자인 시스템을 구축하거나 색상 이론을 다룰 때, 서로 다른 색상 공간 간 변환이 필요합니다. HSL에서 OKLCH로, 또는 HCT에서 HSL로 변환하려면 복잡한 수식을 직접 계산하거나 여러 온라인 도구를 오가며 작업해야 합니다. 게다가 접근성 검증까지 고려하면 작업이 더욱 번거로워집니다.

### 고민거리들
- **여러 도구 왔다 갔다**: HSL → OKLCH는 A 사이트, 접근성 검사는 B 사이트
- **수동 계산의 고통**: 색상 공간 변환 공식을 직접 적용하기 어려움
- **실시간 피드백 부족**: 값을 조정할 때마다 결과를 즉시 확인하기 어려움
- **접근성 고려 누락**: WCAG 대비율을 확인하려면 추가 단계가 필요

### 누가 이런 문제를 겪나요?
색상 이론에 관심 있는 프론트엔드 개발자, 디자인 시스템을 구축하는 팀, 접근성을 고려한 색상 팔레트를 만드는 디자이너들

## 💡 해결책

### 핵심 아이디어
HSL, HCT, OKLCH 세 가지 색상 공간을 실시간으로 변환하고 시각화하는 웹 도구입니다. 하나의 값을 조정하면 다른 색상 공간의 값들이 즉시 업데이트되고, 동시에 WCAG 접근성 대비율도 자동으로 계산됩니다. 복잡한 색상 이론을 직관적인 인터페이스로 만들어 실무에서 바로 활용할 수 있게 만듭니다.

### 작동 방식
1. **색상 입력**: 세 가지 색상 공간 중 아무 곳에나 값 입력
2. **실시간 변환**: 다른 두 색상 공간의 값이 자동 계산되어 표시
3. **시각적 확인**: 변환된 색상을 실제로 화면에서 미리보기
4. **접근성 검증**: 선택한 배경색과 텍스트 색의 WCAG AA/AAA 준수 여부 표시

### 주요 기능
- **삼각 동기화**: HSL ↔ HCT ↔ OKLCH 실시간 양방향 변환
- **슬라이더 인터페이스**: 각 색상 공간의 값을 직관적으로 조정
- **접근성 점수판**: WCAG 2.1 AA/AAA 기준 대비율 자동 계산
- **색상 히스토리**: 최근 사용한 색상 조합을 저장하고 재사용
- **CSS 코드 생성**: 선택한 색상을 CSS 변수나 utility 클래스로 내보내기

### 혁신 포인트

- 접근성 검증을 색상 선택과 동시에 진행
- CSS-in-JS와 디자인 토큰 트렌드에 맞춘 코드 생성

### 학습 기회
- 색상 공간 변환 수학과 알고리즘 이해
- 최신 CSS 색상 스펙 (CSS Color Module Level 4) 실무 적용
- 접근성 기준을 코드로 구현하는 방법

### 왜 이런 공백이 존재하는가
OKLCH와 HCT는 비교적 새로운 색상 공간이라 아직 통합적인 도구가 부족합니다. 특히 한국어 UI로 색상 이론을 쉽게 설명하면서 실무에서 바로 쓸 수 있는 도구는 거의 없습니다.

### 성공 지표
- [ ] 세 색상 공간 간 변환 정확도 99% 이상
- [ ] 슬라이더 조작 시 16ms 이하 응답 시간
- [ ] WCAG 대비율 계산 결과가 WebAIM 도구와 일치
