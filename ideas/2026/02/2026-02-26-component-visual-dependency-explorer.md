# 컴포넌트 시각적 의존성 탐색기

> React 컴포넌트들이 어떻게 연결되어 있는지 인터랙티브한 시각화로 한눈에 파악하기

## 🎯 문제 정의

### 현재 상황
대형 React 프로젝트에서 컴포넌트 간의 의존성을 파악하는 것은 마치 미로를 탐험하는 것과 같습니다. 개발자들은 하나의 기능을 이해하기 위해 수십 개의 파일을 오가며 정신적 지도를 그려야 합니다. 기존의 도구들은 정적인 텍스트 기반 분석이나 단순한 트리 구조만 제공할 뿐입니다.

### 주요 고충
- **정신적 과부하**: 하나의 컴포넌트를 이해하기 위해 20개 이상의 파일을 동시에 열어둠
- **관계 추적의 어려움**: 어떤 컴포넌트가 어디서 사용되는지, 어떤 props를 전달받는지 파악하기 어려움
- **영향도 분석 불가**: 컴포넌트를 수정했을 때 어떤 부분에 영향을 미칠지 예측하기 어려움
- **문서화 부족**: 컴포넌트 간의 관계가 코드에만 존재하고 시각적 문서가 없음

### 누가 이 문제를 겪는가?
- 100개 이상의 컴포넌트를 가진 React 프로젝트에서 작업하는 프론트엔드 개발자
- 새로운 팀원으로 대형 코드베이스에 온보딩하는 개발자
- 레거시 React 프로젝트를 리팩토링해야 하는 시니어 개발자

## 💡 해결책

### 핵심 개념
컴포넌트 시각적 의존성 탐색기는 React 프로젝트의 컴포넌트 구조를 인터랙티브한 그래프로 시각화합니다. 각 컴포넌트는 노드로, props와 import 관계는 연결선으로 표현되어 전체 아키텍처를 한눈에 파악할 수 있습니다. D3.js의 force-directed 레이아웃을 활용해 관련성이 높은 컴포넌트들을 자동으로 클러스터링하고, 실시간으로 탐색할 수 있습니다.

### 작동 방식
1. **프로젝트 분석**: AST 파서가 React 프로젝트를 스캔해서 컴포넌트와 의존성 정보 추출
2. **그래프 생성**: 컴포넌트를 노드로, 의존성을 엣지로 하는 그래프 데이터 구조 생성
3. **시각화 렌더링**: D3.js로 force-directed 그래프를 렌더링하고 인터랙션 기능 추가
4. **실시간 탐색**: 노드 클릭, 필터링, 경로 추적 등의 탐색 기능 제공

### 핵심 기능
- **인터랙티브 그래프**: 드래그, 줌, 필터링이 가능한 컴포넌트 의존성 그래프
- **경로 추적**: 특정 컴포넌트에서 다른 컴포넌트까지의 의존성 경로를 하이라이트
- **Props 플로우 시각화**: 어떤 props가 어떤 경로로 전달되는지 애니메이션으로 표시
- **클러스터 분석**: 관련성이 높은 컴포넌트 그룹을 자동으로 식별하고 색상으로 구분
- **실시간 업데이트**: 파일 변경 감지해서 그래프를 실시간으로 업데이트

## 🏗️ 기술 아키텍처

### 시스템 개요
```
┌─────────────────┐
│   AST Parser    │ (코드 분석)
└────────┬────────┘
         │
┌────────▼────────┐
│ Dependency      │ (의존성 그래프)
│ Graph Builder   │
└────────┬────────┘
         │
┌────────▼────────┐
│ D3.js Renderer  │ (시각화)
└────────┬────────┘
         │
┌────────▼────────┐
│ React UI        │ (컨트롤 패널)
└─────────────────┘
```

### 기술 스택 선택 이유
- **TypeScript**: 복잡한 그래프 데이터 구조와 AST 처리에서 타입 안정성 보장
- **D3.js**: Force-directed 레이아웃과 복잡한 시각화를 위한 최적의 라이브러리
- **AST Parser (ts-morph)**: TypeScript/JSX 코드를 정확하게 분석하기 위한 검증된 도구
- **React**: 컨트롤 패널과 UI 컴포넌트를 위한 익숙한 프레임워크
- **Vite**: 빠른 개발 환경과 hot reload를 위한 모던 번들러

### 주요 기술적 도전과제
1. **대용량 그래프 렌더링**: 500개 이상의 컴포넌트를 가진 프로젝트에서도 부드러운 인터랙션 보장. Canvas 기반 렌더링과 가상화 기법 적용 필요.
2. **정확한 의존성 추출**: 동적 import, conditional rendering, HOC 등 복잡한 React 패턴들을 정확히 분석하는 알고리즘 구현.

## 🎨 이 프로젝트의 흥미로운 점

### 혁신 포인트
- React 생태계에 특화된 의존성 시각화 (props, context, hooks 관계까지 포함)
- 실시간 코드 변경 감지와 그래프 업데이트
- 컴포넌트 "건강도" 지표 (순환 의존성, 과도한 props 등을 자동 감지)
- 게임 UI에서 영감을 받은 직관적인 탐색 인터페이스

### 학습 기회
- D3.js의 고급 시각화 기법과 force simulation 알고리즘
- AST 분석을 통한 정적 코드 분석 기법
- 대용량 데이터의 효율적인 렌더링과 성능 최적화
- React 생태계의 깊은 이해 (컴포넌트 패턴, 상태 관리 등)
