# MCP 서버 자동 배치 도구

> MCP 서버들을 하나의 설정 파일로 자동 배치하고 관리하는 CLI 도구

## 🎯 문제 상황

### 현재 상황
Model Context Protocol(MCP) 서버들이 늘어나면서 각각 다른 설정, 환경변수, 의존성을 가지게 되었습니다. 개발자들은 프로젝트마다 여러 MCP 서버를 수동으로 설정하고 관리해야 합니다. 각 서버의 설치, 설정, 실행 과정이 모두 달라 일관성 있는 관리가 어렵습니다.

### 고통 포인트
- **반복적인 수동 설정**: 새 프로젝트마다 동일한 MCP 서버들을 하나씩 설정
- **환경별 설정 차이**: 개발/스테이징/프로덕션 환경마다 다른 설정 파일들
- **의존성 지옥**: 각 MCP 서버의 서로 다른 Node.js 버전 요구사항
- **상태 추적 불가능**: 어떤 서버가 실행중인지, 실패했는지 파악 어려움

### 누가 겪고 있는가?
MCP 생태계를 적극 활용하는 개발자들, 특히 여러 프로젝트에서 동일한 MCP 서버 조합을 사용하는 팀들과 개인 개발자들입니다.

## 💡 해결책

### 핵심 개념
MCP Deploy는 단일 YAML 설정 파일로 여러 MCP 서버의 배치, 실행, 모니터링을 자동화하는 CLI 도구입니다. Docker를 활용해 각 서버를 격리된 환경에서 실행하며, 의존성 충돌을 방지합니다. 설정 파일 하나로 전체 MCP 서버 스택을 버전 관리하고 재현 가능하게 만듭니다.

### 작동 원리
1. **설정 파일 읽기**: `mcp-deploy.yml`에서 서버 목록과 설정 로드
2. **자동 컨테이너화**: 각 MCP 서버를 Docker 컨테이너로 빌드 및 실행
3. **네트워킹 설정**: 서버간 통신이 필요한 경우 자동으로 네트워크 구성
4. **상태 모니터링**: 실행중인 서버들의 상태를 실시간 추적 및 보고

### 주요 기능
- **선언적 설정**: YAML 파일로 모든 MCP 서버와 설정을 정의
- **환경별 오버라이드**: 개발/프로덕션 환경에 맞는 설정 자동 적용
- **의존성 격리**: Docker 컨테이너로 각 서버의 런타임 환경 완전 분리
- **헬스 체크**: 자동 상태 모니터링과 실패한 서버 재시작
- **로그 통합**: 모든 MCP 서버 로그를 중앙집중식으로 수집 및 필터링

## 📊 시장 및 검증

### 유사 프로젝트
- **Docker Compose**: 범용 컨테이너 오케스트레이션. MCP 서버에 특화된 기능이 없음
- **PM2**: Node.js 프로세스 매니저. 컨테이너화와 MCP 특화 기능 부재
- **Kubernetes**: 너무 복잡하고 과도한 기능, 개발 환경에 부적합

### 이 틈새가 존재하는 이유
MCP는 비교적 새로운 프로토콜이고, 서버 생태계가 빠르게 성장하고 있어 전용 관리 도구의 필요성이 대두되고 있습니다. 기존 도구들은 MCP 서버의 특수한 요구사항(프로토콜 헬스 체크, 컨텍스트 관리 등)을 고려하지 않습니다.

### 성공 지표
- [ ] 5개 이상의 서로 다른 MCP 서버를 동시 관리 가능
- [ ] 환경별 설정 오버라이드가 정상 작동
- [ ] 개발자가 1분 내에 전체 MCP 스택을 실행 가능
