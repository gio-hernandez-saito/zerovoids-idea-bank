# OKLCH 색상 팔레트 생성기

> 지각적으로 균등한 OKLCH 색상 공간을 활용해 접근성이 보장된 색상 팔레트를 직관적으로 생성하는 도구

## 🎯 문제점

### 현재 상황
대부분의 디자이너와 개발자는 HSL이나 RGB 색상 공간을 사용하여 색상 팔레트를 만듭니다. 하지만 이러한 색상 공간은 인간의 색상 지각과 일치하지 않아 일관된 밝기나 채도의 색상을 만들기 어렵습니다. 접근성 가이드라인(WCAG)을 준수하는 색상 조합을 찾는 것은 더욱 복잡한 작업입니다.

### 문제점
- **지각적 불일치**: HSL에서 같은 밝기(Lightness) 값이라도 색상에 따라 실제 밝기가 다르게 보임
- **접근성 검증의 어려움**: 색상 조합이 WCAG 기준을 만족하는지 일일이 확인해야 함
- **일관성 없는 팔레트**: 색조가 다른 색상들 간 시각적 균형이 맞지 않음
- **복잡한 도구**: 기존 색상 도구들은 OKLCH를 지원하지 않거나 사용법이 복잡함

### 누가 이 문제를 겪나요?
UI/UX 디자이너, 프론트엔드 개발자, 디자인 시스템을 구축하는 팀들. 특히 접근성을 중시하는 제품을 개발하는 모든 사람들이 이 문제에 직면합니다.

## 💡 해결책

### 핵심 개념
OKLCH 색상 팔레트 생성기는 지각적으로 균등한 OKLCH 색상 공간을 활용하여 일관된 밝기와 채도를 가진 색상 팔레트를 생성합니다. 실시간으로 접근성 대비비를 계산하고, 시각적으로 직관적인 3D 색상 공간 탐색을 제공합니다. 생성된 팔레트는 다양한 포맷(CSS, JSON, Figma 토큰)으로 내보낼 수 있습니다.

### 작동 방식
1. **OKLCH 공간 탐색**: 3D 원통형 좌표계에서 L(밝기), C(채도), H(색조) 축을 직관적으로 조작
2. **자동 대비비 검증**: 선택한 색상들이 WCAG AA/AAA 기준을 만족하는지 실시간 확인
3. **팔레트 자동 생성**: 기준 색상에서 조화로운 색상들을 자동으로 생성하거나 수동으로 조정
4. **내보내기**: CSS 변수, 디자인 토큰, Figma 플러그인 등 다양한 형태로 출력

### 주요 기능
- **3D 색상 공간 시각화**: OKLCH 원통형 좌표계를 직관적인 3D 인터페이스로 표현
- **실시간 접근성 검증**: 텍스트와 배경 조합의 대비비를 즉시 확인하고 WCAG 등급 표시
- **지능형 팔레트 생성**: 하나의 기준 색상에서 조화로운 색조, 단색조, 보색 팔레트 자동 생성
- **다양한 내보내기 형식**: CSS 커스텀 속성, JSON, Figma 토큰, Adobe 스와치 등 지원

2. **3D 인터페이스 최적화**: 수백 개의 색상 점을 부드럽게 렌더링하면서 60fps 유지

### 혁신 포인트

- 디자인 시스템 워크플로우에 최적화된 내보내기 기능
- 색상 이론 교육적 가치 (OKLCH 개념을 직관적으로 학습)

### 학습 기회
- OKLCH 색상 공간과 지각적 균등성 개념 이해
- Three.js를 활용한 3D 데이터 시각화 기법
- 웹 접근성과 색상 대비비 계산 알고리즘
- 복잡한 수학적 변환의 실시간 처리 최적화

## 📊 시장 및 검증

### 유사 프로젝트
- **Coolors.co**: HSL 기반 팔레트 생성기. OKLCH 지원 없음, 3D 탐색 불가
- **Adobe Color**: 전문적이지만 복잡함. OKLCH 미지원
- **Accessible Colors**: 접근성 검증에 특화되어 있으나 색상 생성 기능 제한적

### 이 틈새가 존재하는 이유
OKLCH는 상대적으로 새로운 색상 공간이고, 브라우저 지원도 최근에야 안정화되었습니다. 기존 도구들은 HSL/RGB에 최적화되어 있어 새로운 색상 공간을 지원하려면 상당한 재개발이 필요합니다. 또한 3D 시각화는 기술적 복잡도가 높아 시도되지 않았습니다.

### 성공 지표
- [ ] OKLCH → RGB 변환이 1ms 이내에 완료
- [ ] 100개 색상 포인트를 60fps로 부드럽게 렌더링
- [ ] WCAG 대비비 계산 정확도 99.9%
- [ ] 생성된 팔레트를 5가지 이상 형식으로 내보내기 가능
