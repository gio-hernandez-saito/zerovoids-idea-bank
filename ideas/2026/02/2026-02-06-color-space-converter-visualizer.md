# 색공간 변환 시각화 도구

> HSL, OKLCH, HCT 간 실시간 변환을 시각적으로 보여주며 접근성을 검증하는 인터랙티브 도구

## 🎯 문제점

### 현재 상황
현대 웹 개발에서 색상 관리는 점점 복잡해지고 있습니다. HSL, OKLCH, HCT 같은 다양한 색공간이 등장했지만, 이들 간의 관계와 변환 과정을 직관적으로 이해하기 어렵습니다. 디자인 시스템을 구축할 때 색상 접근성을 고려해야 하지만, 현재 도구들은 단순한 변환만 제공할 뿐입니다.

### 주요 불편사항
- **색공간 간 차이 이해 부족**: HSL의 lightness와 OKLCH의 lightness가 시각적으로 어떻게 다른지 모름
- **접근성 검증의 어려움**: 색상 조합이 WCAG, APCA 기준에서 어느 정도인지 즉석에서 확인 불가
- **디자인 토큰 생성의 복잡성**: 일관된 색상 팔레트를 체계적으로 만들기 어려움

### 누가 겪는 문제인가?
디자인 시스템을 구축하는 프론트엔드 개발자, 색상 접근성을 고려해야 하는 UI/UX 디자이너, 그리고 색상 이론을 깊이 이해하고 싶은 개발자들입니다.

## 💡 해결방안

### 핵심 개념
색공간 변환 시각화 도구는 여러 색공간을 동시에 보여주는 인터랙티브 환경입니다. 하나의 색상을 조정하면 HSL, OKLCH, HCT 표현이 실시간으로 업데이트되며, 각 색공간의 특성을 3D 시각화로 보여줍니다. 동시에 선택한 색상의 접근성 점수를 WCAG와 APCA 기준으로 계산하여 표시합니다.

### 작동 방식
1. **색상 선택**: 컬러 피커나 값 입력으로 기준 색상 설정
2. **실시간 변환**: 모든 색공간 표현이 동시에 업데이트됨
3. **3D 시각화**: 각 색공간의 3차원 구조 내에서 현재 색상 위치 표시
4. **접근성 분석**: 배경/텍스트 조합의 접근성 점수 자동 계산

### 주요 기능
- **멀티 색공간 뷰어**: HSL, OKLCH, HCT를 동시에 표시하고 실시간 동기화
- **3D 색공간 시각화**: Three.js로 구현한 색공간의 3차원 구조 탐색
- **접근성 검증기**: WCAG 2.1, WCAG 3.0 (APCA) 기준으로 대비율 분석
- **팔레트 생성기**: 색조 기반으로 일관된 색상 스케일 자동 생성
