# 야구 타순 시뮬레이터

> 타순 변경이 경기 결과에 미치는 영향을 실시간으로 시뮬레이션하는 인터랙티브 도구

## 🎯 문제 상황

### 현재 상황
야구에서 타순은 전략의 핵심이지만, 다른 타순으로 했을 때의 결과를 알기는 불가능합니다. 감독들은 직감과 경험에 의존하지만, "만약 이 선수를 3번으로 올렸다면?"과 같은 가정은 검증하기 어렵습니다. 기존 야구 통계 사이트들은 과거 데이터만 보여줄 뿐, 가상의 시나리오를 탐색할 수 없습니다.

### 문제점
- **가정 검증 불가**: "만약에"에 대한 답을 구하기 어려움
- **타순 최적화의 어려움**: 수많은 조합 중 최적해를 찾기 복잡
- **시각적 이해 부족**: 타순 변경의 파급효과를 직관적으로 파악하기 어려움
- **실시간 분석 불가**: 경기 중 타순 변경 효과를 즉시 확인할 수 없음

### 누가 이런 문제를 겪나요?
야구 팬, 판타지 야구 플레이어, 아마추어 팀 감독, 야구 분석가, 그리고 데이터로 야구를 이해하고 싶어하는 사람들입니다.

## 💡 해결 방안

### 핵심 아이디어
야구 타순 시뮬레이터는 실제 선수 데이터를 기반으로 다양한 타순 조합의 예상 득점을 시뮬레이션합니다. 사용자가 타순을 드래그 앤 드롭으로 변경하면, 몬테카르로 시뮬레이션을 통해 1000번의 가상 경기를 실행하고 결과를 실시간으로 시각화합니다. 단순한 통계가 아닌, 타순 변경이 이닝별, 상황별로 어떤 영향을 미치는지 보여줍니다.

### 작동 방식
1. **팀 구성**: 실제 선수 데이터로 9명의 타자 선택
2. **타순 설정**: 드래그 앤 드롭으로 타순 조정
3. **시뮬레이션 실행**: 1000경기 몬테카르로 시뮬레이션 자동 실행
4. **결과 시각화**: D3.js로 득점 분포, 이닝별 효과, 상황별 분석 표시

### 주요 기능
- **실시간 타순 편집기**: 드래그 앤 드롭으로 타순 변경, 즉시 시뮬레이션 재실행
- **다차원 결과 분석**: 총 득점, 이닝별 득점, 주자 상황별 효과를 동시에 시각화
- **타순 최적화 추천**: 유전 알고리즘으로 최적 타순 조합 제안
- **비교 모드**: 여러 타순을 동시에 비교하여 장단점 분석

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────┐
│ 선수 데이터 API │ (실제 타율, OPS 등)
└────────┬────────┘
         │
┌────────▼────────┐
│ 시뮬레이션 엔진 │ (몬테카르로)
└────────┬────────┘
         │
┌────────▼────────┐
│ 결과 분석기     │ (통계 계산)
└────────┬────────┘
         │
┌────────▼────────┐
│ D3 시각화       │ (인터랙티브 차트)
└────────┬────────┘
         │
┌────────▼────────┐
│ React UI        │ (타순 편집기)
└─────────────────┘
```

### 기술 스택 선택 이유
- **React + TypeScript**: 복잡한 상태 관리(타순, 시뮬레이션 결과)를 타입 안전하게 처리
- **D3.js**: 다차원 데이터(득점분포, 이닝별 효과)의 복합 시각화에 최적
- **Web Workers**: 몬테카르로 시뮬레이션을 백그라운드에서 실행하여 UI 블로킹 방지
- **Vite**: 빠른 개발 환경과 최적화된 번들링

### 주요 기술 과제
1. **시뮬레이션 성능 최적화**: 1000경기 시뮬레이션을 1초 내에 완료하기 위한 알고리즘 최적화
2. **복합 데이터 시각화**: 득점 분포, 이닝별 효과, 상황별 분석을 한 화면에 직관적으로 표현
3. **실시간 업데이트**: 타순 변경 시 부분적 재계산을 통한 빠른 응답성 구현
