# Git 히스토리 3D 시각화 도구

> Git 커밋 히스토리를 3D 공간에서 탐험하며 프로젝트 진화 과정을 직관적으로 이해하는 도구

## 🎯 문제 정의

### 현재 상황
Git 히스토리를 이해하는 것은 개발자에게 필수적이지만, 기존 도구들은 2D 그래프나 텍스트 기반 로그에 의존합니다. 복잡한 브랜치 구조나 장기간의 개발 히스토리를 파악하기 어려우며, 시간의 흐름에 따른 코드 변화를 직관적으로 이해하기 힘듭니다.

### 주요 문제점
- **평면적 시각화의 한계**: 2D 그래프로는 복잡한 브랜치 관계를 표현하기 어려움
- **시간감각 부족**: 커밋 간의 시간 간격과 개발 속도를 직관적으로 파악 불가능
- **컨텍스트 손실**: 코드 변경량, 작성자, 관련 파일 정보가 분리되어 표시됨
- **탐험적 분석 부족**: 특정 기간이나 기능의 진화 과정을 자유롭게 탐색하기 어려움

### 누가 이 문제를 겪나요?
복잡한 프로젝트에 새로 투입된 개발자, 레거시 코드를 분석해야 하는 시니어 개발자, 프로젝트 진행 상황을 파악해야 하는 팀 리더들이 주로 겪습니다.

## 💡 해결책

### 핵심 컨셉
Git 히스토리를 3차원 공간에 시각화하여 시간, 브랜치, 변경량을 직관적으로 표현하는 도구입니다. 각 커밋은 3D 공간의 노드로 표현되며, 시간축을 따라 배치됩니다. 브랜치는 서로 다른 색상과 높이로 구분되고, 커밋의 크기는 변경량을 반영합니다.

### 작동 방식
1. **저장소 분석**: Git 히스토리를 파싱하여 커밋, 브랜치, 파일 변경 정보 추출
2. **3D 레이아웃 생성**: 시간축을 기준으로 커밋들을 3D 공간에 배치, 브랜치별로 층을 분리
3. **인터랙티브 탐색**: 마우스나 키보드로 자유롭게 시점 이동, 특정 커밋이나 기간에 줌인
4. **상세 정보 표시**: 커밋 클릭 시 변경 파일, 작성자, 메시지 등 상세 정보 오버레이

### 주요 기능
- **시간 기반 3D 레이아웃**: Z축을 시간으로, Y축을 브랜치로 사용하여 직관적 배치
- **동적 필터링**: 작성자, 날짜 범위, 파일 타입별 필터링으로 원하는 정보만 표시
- **애니메이션 플레이백**: 프로젝트 진화 과정을 시간 순으로 재생하는 기능
- **브랜치 관계 시각화**: Merge, rebase 관계를 3D 연결선으로 명확히 표현

## 🏗️ 기술 아키텍처

### 시스템 구조
```
┌─────────────────┐
│  Git Parser     │ (libgit2/nodegit)
└────────┬────────┘
         │
┌────────▼────────┐
│  Data Model     │ (커밋 그래프 구조)
└────────┬────────┘
         │
┌────────▼────────┐
│  Layout Engine  │ (3D 좌표 계산)
└────────┬────────┘
         │
┌────────▼────────┐
│  Three.js       │ (3D 렌더링)
└────────┬────────┘
         │
┌────────▼────────┐
│  React UI       │ (컨트롤 인터페이스)
└─────────────────┘
```

### 기술 스택 선택 이유
- **Three.js**: WebGL 기반 3D 렌더링으로 복잡한 Git 히스토리도 부드럽게 표현 가능
- **TypeScript**: 복잡한 Git 데이터 구조를 타입 안전하게 다루기 위해 필수
- **Node.js + libgit2**: 서버사이드에서 Git 저장소를 효율적으로 분석
- **React**: UI 컨트롤과 필터링 인터페이스를 컴포넌트 단위로 관리

### 주요 기술적 도전
1. **대용량 히스토리 처리**: 수만 개의 커밋을 가진 저장소에서도 부드러운 렌더링을 위한 LOD(Level of Detail) 시스템 구현
2. **3D 레이아웃 알고리즘**: 브랜치 간 충돌을 피하면서도 시간적 연속성을 유지하는 배치 알고리즘 설계
3. **실시간 상호작용**: 줌, 회전, 필터링 시에도 60fps를 유지하는 성능 최적화

- **대용량 데이터 시각화**: 성능과 가독성을 동시에 만족하는 시각화 기법
