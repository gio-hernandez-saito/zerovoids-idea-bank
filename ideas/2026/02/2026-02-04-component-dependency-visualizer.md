# 컴포넌트 의존성 시각화 도구

> React 컴포넌트들의 복잡한 의존성을 직관적인 그래프로 탐험하세요

## 🎯 문제 상황

### 현재 상황
대규모 React 프로젝트에서 컴포넌트 간 의존성을 파악하는 것은 마치 미로를 헤매는 것과 같습니다. 개발자들은 하나의 컴포넌트를 수정할 때 어떤 다른 컴포넌트들이 영향을 받는지 알기 위해 수십 개의 파일을 뒤져야 합니다. 현재의 IDE나 도구들은 단순한 "find usages" 기능만 제공할 뿐, 전체적인 아키텍처 그림을 보여주지 못합니다.

### 주요 문제점
- **정신적 부담 과부하**: 하나의 기능을 이해하기 위해 20개 이상의 파일을 오가며 확인해야 함
- **리팩토링 공포**: 컴포넌트 수정 시 어디서 오류가 날지 예측하기 어려움
- **순환 의존성 발견 지연**: 복잡한 import 관계에서 circular dependency를 찾기 힘듦
- **아키텍처 이해 부족**: 새로운 팀원이 프로젝트 구조를 파악하는데 오랜 시간 소요

### 이 문제를 겪는 사람들
React 기반 모노레포나 100개 이상의 컴포넌트를 가진 프로젝트에서 작업하는 프론트엔드 개발자들, 특히 코드 리뷰나 리팩토링을 담당하는 시니어 개발자들

## 💡 해결책

### 핵심 개념
컴포넌트 의존성 시각화 도구는 React 프로젝트의 컴포넌트들을 분석하여 상호작용하는 그래프로 변환합니다. D3.js의 force-directed layout을 사용해 자연스러운 클러스터링을 만들고, 각 컴포넌트를 노드로, import 관계를 엣지로 표현합니다. 사용자는 특정 컴포넌트를 클릭하여 그것의 영향 범위를 즉시 확인하거나, 전체 아키텍처의 "핫스팟"을 한눈에 파악할 수 있습니다.

### 작동 원리
1. **프로젝트 스캔**: TypeScript AST 파서를 통해 모든 React 컴포넌트와 import 관계를 분석
2. **그래프 생성**: 의존성 데이터를 기반으로 방향성 그래프 구조를 구축
3. **인터랙티브 시각화**: D3.js로 force-directed layout을 렌더링하고 실시간 상호작용 제공
4. **분석 및 인사이트**: 순환 의존성, 고립된 컴포넌트, 과도한 결합도 등을 자동 감지 및 하이라이트

### 주요 기능
- **실시간 의존성 탐색**: 컴포넌트 클릭 시 직접/간접 의존성을 시각적으로 하이라이트
- **아키텍처 레이어 감지**: 자동으로 presentation, container, utility 컴포넌트를 분류하고 색상 구분
- **순환 의존성 경고**: 문제가 되는 circular import를 빨간색으로 강조 표시
- **영향도 분석**: 특정 컴포넌트 변경 시 영향받는 모든 컴포넌트를 시뮬레이션
- **필터링 및 검색**: 특정 패턴이나 폴더별로 그래프를 필터링하여 집중 분석 가능

## 🏗️ 기술 아키텍처

### 시스템 개요
```
┌─────────────────┐
│  AST Parser     │ (TypeScript/React 코드 분석)
└────────┬────────┘
         │
┌────────▼────────┐
│ Dependency      │ (그래프 데이터 구조)
│ Graph Builder   │
└────────┬────────┘
         │
┌────────▼────────┐
│ D3.js Force     │ (시각화 엔진)
│ Layout Engine   │
└────────┬────────┘
         │
┌────────▼────────┐
│ React UI        │ (컨트롤 패널 & 상호작용)
│ Components      │
└─────────────────┘
```

### 기술 스택 선택 이유
- **TypeScript AST Parser (ts-morph)**: React 컴포넌트와 import 구문을 정확하게 파싱하고, 타입 정보도 함께 추출 가능
- **D3.js**: Force-directed graph의 물리 시뮬레이션과 부드러운 애니메이션을 제공하여 직관적인 레이아웃 생성
- **React + TypeScript**: 복잡한 UI 상태 관리와 타입 안정성을 보장하면서 컴포넌트 기반 아키텍처 구현
- **Vite**: 빠른 개발 서버와 HMR로 반복 개발 속도 향상

### 주요 기술적 도전과제
1. **대규모 그래프 성능 최적화**: 1000개 이상의 컴포넌트를 부드럽게 렌더링하기 위한 가상화와 LOD(Level of Detail) 시스템 구현
2. **의존성 순환 감지 알고리즘**: DFS 기반 cycle detection을 최적화하여 실시간으로 순환 의존성을 찾아내는 효율적인 방법

## 🎨 이 프로젝트의 독특한 점

### 혁신 포인트
- React 컴포넌트 아키텍처를 게임의 관계 맵처럼 탐험할 수 있는 경험
- Force-directed layout과 React 생태계의 결합으로 자연스러운 컴포넌트 클러스터링
- AST 분석과 실시간 시각화를 통한 "코드를 보는 새로운 방식"
- 개발자의 인지 부하를 줄이는 공간적 메모리 활용

### 학습 기회
- D3.js의 force simulation과 복잡한 데이터 바인딩 패턴
- TypeScript AST 조작과 코드 정적 분석 기법
- 대규모 데이터 시각화에서의 성능 최적화 전략
