# 별자리 날씨 예보 시각화 도구

> 7일간의 날씨 예보를 밤하늘 별자리처럼 펼쳐놓은 인터랙티브 시각화 — 기상 데이터가 우주 지도가 된다

## 🎯 문제 정의

### 현재 상황
일반적인 날씨 앱은 숫자와 아이콘의 나열이다. 기온, 강수 확률, 습도, 바람 세기가 테이블 형식으로 제공되지만, 여러 변수들이 어떻게 서로 연관되는지, 시간 흐름에 따른 패턴이 어떻게 형성되는지를 한눈에 파악하기 어렵다. 특히 야외 활동(등산, 캠핑, 천체 관측)을 계획하는 사람들은 단순한 숫자 이상의 정보를 필요로 한다.

### 불편한 점
- **다변수 파악 어려움**: 기온·습도·바람·강수량이 각각 분리되어 있어 종합적 판단이 어렵다
- **패턴 인식 부재**: 주간 날씨 흐름이 숫자 나열로 제시되어 고점·저점·급변 구간을 직관적으로 볼 수 없다
- **천체 관측 계획의 불편함**: 맑은 하늘이 언제인지, 달이 언제 뜨는지 등 복합 조건을 여러 앱을 조합해야 확인할 수 있다

### 대상 사용자
천체 관측 애호가, 야외 활동 계획자, 날씨 데이터를 창의적으로 탐색하고 싶은 사람, 데이터 시각화에 관심 있는 개발자.

## 💡 해결책

### 핵심 개념
현재 위치의 7일 날씨 예보 데이터를 별자리 지도처럼 렌더링한다. 각 시간대(또는 날짜)는 하나의 "별"이 되고, 별의 크기는 강수 확률, 밝기(투명도)는 구름 양, 색상은 기온, 별들 사이의 연결선 굵기는 바람 세기를 나타낸다. 결과적으로 맑고 따뜻한 날은 밝고 큰 별로, 흐리고 비 오는 날은 어둡고 작은 별로 표현되어 — 마치 밤하늘을 보듯 — 한 화면에서 일주일 날씨의 전체 패턴을 파악할 수 있다.

### 동작 방식
1. **위치 감지**: 브라우저 Geolocation API로 현재 위치 좌표 획득 (또는 도시 검색)
2. **날씨 수집**: Open-Meteo API (무료, API 키 불필요)로 시간별 기온·강수 확률·구름·바람 데이터 수집
3. **별자리 매핑**: 각 시간 포인트를 D3.js force simulation으로 원형 배치하고, 각 기상 변수를 시각 속성(크기·색·투명도·연결선)으로 인코딩
4. **인터랙션**: 별 위에 마우스를 올리면 툴팁으로 상세 수치 표시, 클릭하면 해당 시간대 확대
5. **천문학 레이어**: 선택적으로 해당 날짜 실제 별자리 오버레이 토글 가능 (NASA Horizons API 또는 정적 별 데이터)

### 주요 기능
- **다변수 동시 인코딩**: 4개 기상 변수를 단일 시각화에 통합 표현
- **천문학 오버레이**: 실제 밤하늘 별자리와 날씨 시각화의 레이어 전환
- **최적 관측 시간 하이라이트**: 구름 적고, 바람 잔잔하고, 달이 없는 구간 자동 표시
- **반응형 Canvas**: SVG와 Canvas를 혼합하여 수백 개 포인트도 부드럽게 렌더링

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌──────────────────────────────────────────────┐
│  사용자 브라우저                              │
│                                              │
│  Geolocation API → 좌표 획득                 │
│         │                                    │
│  Open-Meteo API → 시간별 기상 JSON           │
│         │                                    │
│  DataMapper → 기상값을 시각 속성으로 변환    │
│         │                                    │
│  D3 Force Simulation → 별 배치 계산          │
│         │                                    │
│  Canvas + SVG 렌더러 → 최종 시각화           │
│         │                                    │
│  React UI 레이어 → 컨트롤, 툴팁, 범례        │
└──────────────────────────────────────────────┘
```

### 기술 스택 선택 이유
- **Open-Meteo API**: 무료, API 키 불필요, 시간별 데이터 제공 — 프로토타입에 최적
- **D3.js Force Simulation**: 별 배치를 자동으로 계산, 겹치지 않게 분포
- **Canvas API**: 수십~수백 개 별 애니메이션을 SVG보다 효율적으로 렌더링
- **TypeScript**: 복잡한 데이터 변환 파이프라인에서 타입 안전성 보장

### 핵심 기술 과제
1. **다변수 → 시각 속성 매핑 설계**: 4개 변수를 어떤 시각 채널에 배치해야 인지 부하가 낮은가? (크기·색조·명도·연결선 두께의 사용자 테스트 필요)
2. **Canvas와 D3 통합**: D3는 DOM 기반이지만 Canvas로 렌더링할 때 이벤트 감지를 직접 구현해야 함 (포인트 in circle 히트 테스트)
3. **실시간 애니메이션**: 별이 생성될 때 반짝이는 등장 애니메이션 구현 (requestAnimationFrame + easing)

## 🎨 무엇이 흥미로운가?

### 독창적 관점
날씨 데이터를 천문학 은유로 재해석하는 것은 이전에 없던 조합이다. 기상 앱은 항상 달력 또는 선 그래프 형식을 고수하지만, 이 프로젝트는 "날씨도 별처럼 패턴이 있다"는 아이디어에서 출발한다. 특히 천체 관측 계획에 실제로 유용하다는 점에서 은유가 기능과 일치한다.

### 혁신 포인트
- 기상 데이터의 별자리 은유 — 기능적 비유이지 장식적 비유가 아님
- 4개 기상 변수의 단일 시각화 통합 (대부분 앱은 탭으로 분리)
- 실제 천문학 데이터와 기상 데이터의 레이어 합성
- 천체 관측 "골든 타임" 자동 탐지 기능

### 학습 기회
- D3 Force Simulation의 커스텀 배치 알고리즘
- Canvas와 SVG 혼합 렌더링 기법
- 다변수 데이터 시각 인코딩 원칙 (Mackinlay의 시각 채널 이론)
- 브라우저 Geolocation API와 외부 REST API 통합
- 천문학 각도 계산 (적경·적위 → 화면 좌표)

## 📊 시장 및 검증

### 유사 프로젝트
- **Weather.com / AccuWeather**: 표준 날씨 UI. 다변수 통합 시각화 없음
- **Windy.com**: 풍속·기압 레이어 지도. 별자리 은유와 천문학 통합 없음
- **Clear Outside (천체 관측 앱)**: 관측 조건 특화. 일반 날씨 시각화 미흡

### 이 틈새가 존재하는 이유
날씨 앱은 정보 전달 최적화에 집중하여 시각적 참여도(engagement)를 희생했다. 천체 관측 앱은 기능적이지만 시각적으로 매력적이지 않다. 이 프로젝트는 두 영역의 중간 — 아름답고 정보 밀도 높은 — 을 목표로 한다.

## 🌱 무엇을 배우게 되는가?

### 새로운 기술과 개념
- **D3 Force Simulation**: 자동 배치 알고리즘의 원리 (charge, link, collision 힘)
- **다변수 시각 인코딩**: 어떤 데이터에 어떤 시각 채널을 써야 하는가 (색상 vs 크기 vs 위치)
- **Canvas 히트 테스트**: SVG 없이 마우스 이벤트를 Canvas에서 처리하는 방법
- **REST API 병렬 호출**: Promise.all로 기상·천문 데이터 동시 수집
- **천문학 좌표 변환**: 적경·적위를 화면 2D 좌표로 투영

### 성장 기회
- 데이터 은유 설계 — 데이터를 표현하는 은유를 선택할 때 무엇을 고려해야 하는가
- 정보 밀도와 인지 부하의 균형 — 얼마나 많은 변수를 한 화면에 담을 수 있는가
- Canvas 성능 패턴 내면화 — requestAnimationFrame, dirty flag, 레이어 분리

### 한계 도전 지점
천문학 좌표 데이터를 처음 다루는 영역이라 각도 계산과 투영 수학이 낯선 도전이 될 수 있다. 또한 Canvas 기반 인터랙션은 SVG보다 훨씬 수동적인 구현이 필요하여 이벤트 처리 아키텍처를 직접 설계해야 한다.
