# 게임 이론 기반 오픈소스 기여 전략 시뮬레이터

> 죄수의 딜레마부터 내시 균형까지 — 오픈소스 생태계를 경제학 게임으로 플레이하며 최적 기여 전략을 발견하라

## 🎯 문제

### 현재 상황
오픈소스 생태계는 수천 명의 기여자들이 상호 의존하는 복잡한 협력 시스템이다. "왜 어떤 프로젝트는 번성하고 어떤 프로젝트는 소멸하는가?", "기여자들은 언제 협력하고 언제 무임승차하는가?" 같은 질문은 경험적으로만 이해되어 왔다. 게임 이론이 이 메커니즘을 설명하는 강력한 렌즈를 제공하지만, 대부분의 개발자는 게임 이론을 추상적인 수학 개념으로만 인식한다.

### 페인 포인트
- **추상적 개념의 벽**: 죄수의 딜레마, 내시 균형 같은 게임 이론 개념이 오픈소스 현실과 어떻게 연결되는지 직관적으로 이해하기 어려움
- **전략 최적화의 막막함**: 오픈소스 기여 시 "어떤 프로젝트에, 얼마나, 어떤 방식으로" 기여해야 최대 효과를 내는지 체계적 판단 기준 부재
- **무임승차 문제의 이해 부족**: 왜 훌륭한 오픈소스 프로젝트가 유지보수 번아웃으로 붕괴하는지, 구조적 원인을 파악하는 도구가 없음

### 누가 이 문제를 겪는가?
오픈소스 기여를 시작하려는 개발자, 커뮤니티 전략을 고민하는 오픈소스 메인테이너, 경제학이나 게임 이론을 실용적 맥락에서 배우고 싶은 CS 학습자.

## 💡 솔루션

### 핵심 개념
오픈소스 생태계를 게임 이론의 플레이그라운드로 재구성한 인터랙티브 시뮬레이터다. 사용자는 다양한 게임 이론 시나리오(공공재 게임, 반복 죄수의 딜레마, 진화 안정 전략)를 오픈소스 맥락으로 설정하고, 다양한 에이전트 전략(팃포탯, 항상 협력, 무임승차자)들이 어떻게 상호작용하는지 시각적으로 관찰한다. 단순 관람이 아니라 직접 전략을 설계하고 경쟁시키는 "전략 아레나"를 제공한다.

### 작동 방식
1. **시나리오 선택**: 오픈소스 맥락으로 설정된 게임 이론 시나리오를 선택 (예: "100명의 기여자가 있는 공공재 프로젝트", "두 경쟁 프레임워크의 생태계 전쟁")
2. **전략 설계**: 에이전트의 행동 규칙을 직접 정의 (언제 기여하고, 언제 소비하고, 언제 포크할지)
3. **시뮬레이션 실행**: 수백 라운드의 상호작용이 D3 애니메이션으로 펼쳐지며 생태계가 진화
4. **균형점 분석**: 어떤 전략 조합이 내시 균형에 수렴하는지, 어떤 조건에서 협력 생태계가 붕괴하는지 결과 분석
5. **인사이트 추출**: 실제 오픈소스 기여 전략으로 번역 가능한 시사점 도출

### 주요 기능
- **시나리오 라이브러리**: 공공재 게임, 반복 죄수의 딜레마, 스태그 헌트, 치킨 게임을 오픈소스 맥락으로 큐레이션
- **전략 에디터**: 조건 기반 에이전트 행동 규칙을 코드 없이 설계하는 비주얼 편집기
- **진화 시각화**: 세대를 거치며 전략 분포가 변화하는 D3 애니메이션 (파이 차트 + 플로우 다이어그램)
- **균형 탐지기**: 시뮬레이션이 내시 균형에 수렴했는지 자동 감지 및 설명
- **커스텀 전략 코드**: TypeScript로 고급 전략 직접 구현 가능 (Svelte 컴포넌트로 실시간 결과 반영)

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌──────────────────────────────────────────┐
│            Svelte UI Layer               │
│   시나리오 설정 / 전략 에디터 / 결과 패널  │
└──────────────┬───────────────────────────┘
               │
┌──────────────▼───────────────────────────┐
│         Simulation Engine (TS)           │
│  에이전트 관리 / 라운드 계산 / 균형 탐지  │
└──────────────┬───────────────────────────┘
               │
┌──────────────▼───────────────────────────┐
│       D3.js Visualization Layer          │
│  진화 애니메이션 / 페이오프 매트릭스 뷰   │
└──────────────────────────────────────────┘
```

모든 계산은 브라우저에서 실행 (서버리스). Svelte의 반응형 스토어가 시뮬레이션 상태를 관리하고, D3가 변화를 애니메이션으로 렌더링한다.

### 기술 스택 선택 이유
- **Svelte**: 주간 제약 조건(React 제외) 준수. 반응형 상태 관리가 시뮬레이션 루프에 적합하며, 컴파일 타임 최적화로 빠른 렌더링
- **TypeScript**: 에이전트 전략을 타입 안전하게 정의. `Strategy` 인터페이스로 사용자 커스텀 전략과 내장 전략을 동일하게 처리
- **D3.js**: 진화 과정의 데이터 바인딩과 전환 애니메이션. 포스 시뮬레이션으로 에이전트 네트워크 시각화

### 핵심 기술적 도전
1. **에이전트 전략 샌드박싱**: 사용자가 직접 작성한 TypeScript 전략 코드를 안전하게 실행하는 방법. Web Worker + 제한된 API 노출로 격리
2. **균형 수렴 감지**: 시뮬레이션이 내시 균형에 도달했는지 수학적으로 판단하는 알고리즘. 전략 분포의 분산이 임계값 이하로 안정화될 때 감지
3. **대규모 에이전트 성능**: 1000+ 에이전트의 매 라운드 계산을 60fps 애니메이션과 병행. 계산을 Web Worker로 오프로드하고 결과만 메인 스레드로 전달

## 🌱 배울 것들

### 새로운 스킬과 개념
- **Svelte 반응형 패러다임**: React와 다른 컴파일 기반 반응성, `$:` 레이블과 스토어 패턴의 깊은 이해
- **에이전트 기반 모델링**: 단순한 규칙에서 복잡한 창발적 행동이 나타나는 시뮬레이션 설계 원리
- **게임 이론 수학**: 내시 균형 계산, 파레토 최적성, 진화 안정 전략의 프로그래밍적 구현
- **Web Worker 아키텍처**: 계산 집약적 작업을 UI 스레드와 분리하는 실전 패턴

### 성장 기회
- 경제학 + 소프트웨어 생태계라는 낯선 도메인 조합에서 아이디어를 구체화하는 경험
- 수학적 개념(균형 수렴)을 프로그래밍으로 구현하며 이론과 실践을 연결
- 복잡한 시뮬레이션을 직관적 UI로 추상화하는 UX 설계 역량

### 스트레치 포인트
게임 이론 알고리즘을 처음부터 구현하는 것이 가장 큰 도전. 죄수의 딜레마 페이오프 매트릭스는 단순하지만, 진화 안정 전략의 수렴 판단과 공공재 게임의 다자간 상호작용 계산은 수학적 이해가 필요하다.
