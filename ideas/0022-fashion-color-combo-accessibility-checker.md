# 패션 색상 조합 접근성 검증기

> 옷장 속 색상 조합이 색각 이상 사용자에게 어떻게 보이는지 시뮬레이션하고, 더 포용적인 패션 선택을 안내하는 도구

## 🎯 문제

### 현재 상황
패션 업계의 색상 조합 가이드는 시각적으로 "아름다운" 것을 기준으로 만들어졌다. 하지만 전 세계 인구의 약 8%(남성 기준)가 색각 이상을 갖고 있으며, 이들에게는 유행하는 컬러 팔레트가 전혀 다르게 보인다. 패션 디자이너, 스타일리스트, 의류 쇼핑몰 운영자 모두 이 간극을 인식하지 못한 채 컨텐츠를 제작한다.

### 고통 포인트
- **보이지 않는 문제**: 색각 이상 고객은 상품 설명과 실제 착용 경험이 다르지만 그 이유를 설명하기 어렵다
- **디자이너의 맹점**: 패션 디자이너와 스타일리스트는 색각 이상 시뮬레이션 도구를 쓸 줄 모르거나 패션 맥락에 특화된 도구가 없다
- **접근성 기준의 부재**: WCAG는 웹 UI를 위한 기준이고 패션/의류 색상 조합을 평가하는 맥락화된 기준이 없다

### 누가 겪는가?
패션 브랜드의 비주얼 머천다이저, 패션 블로거, 의류 쇼핑몰 상품 기획자, 그리고 색각 이상이 있는 소비자 본인.

## 💡 해결책

### 핵심 개념
패션 색상 조합 접근성 검증기는 상·하의, 아우터, 액세서리 등 레이어드 의류 색상을 입력하면 다양한 색각 이상 유형(적록색맹, 청황색맹, 전색맹 등)의 시뮬레이션을 즉시 생성한다. WCAG/APCA 기준을 패션 맥락에 맞게 재해석하여 "착장 접근성 점수"를 제공하고, 대안 팔레트를 추천한다.

### 작동 방식
1. **색상 입력**: 헥스 코드 입력, 이미지 업로드 후 색상 픽, 또는 시즌 트렌드 팔레트에서 선택
2. **레이어 구성**: 상의/하의/신발/액세서리로 레이어를 구성하고 각각에 색상 할당
3. **접근성 시뮬레이션**: 4가지 색각 이상 유형으로 실시간 렌더링하여 나란히 비교
4. **점수 및 개선안 제시**: 착장 접근성 점수(A/AA/AAA) 표시 + 비슷한 분위기의 접근성 높은 대안 팔레트 3종 추천

### 주요 기능
- **다층 레이어 미리보기**: 실루엣 기반 패션 일러스트에 색상을 적용하여 실제 착용감 모사
- **색각 이상 4종 시뮬레이션**: Protanopia, Deuteranopia, Tritanopia, Achromatopsia
- **착장 접근성 점수**: 색상 간 대비, 색조 다양성, 명도 차이를 종합한 커스텀 점수 시스템
- **대안 팔레트 생성**: OKLCH 색공간 기반으로 원본의 무드를 유지하면서 접근성을 향상시킨 3가지 대안
- **시즌 트렌드 팔레트 라이브러리**: 팬톤 트렌드 색상 등 유행 팔레트를 접근성 점수와 함께 제공

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────────┐
│   Color Input Layer  │ (헥스 입력, 이미지 픽커, 팔레트 선택)
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│  Color Model (OKLCH) │ (Culori 라이브러리로 색공간 변환)
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│  CVD Simulator       │ (색각 이상 행렬 변환 알고리즘)
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│  Accessibility Score │ (커스텀 패션 접근성 점수 계산)
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│  Alternative Gen     │ (OKLCH 공간에서 접근성 개선 팔레트 탐색)
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│  Svelte UI           │ (인터랙티브 패션 미리보기, 나란히 비교)
└─────────────────────┘
```

### 기술 스택 선정 이유
- **Culori**: OKLCH, OKLAB 등 지각적으로 균일한 색공간 변환을 정확하게 처리. 색각 이상 시뮬레이션 행렬 적용에 최적
- **Svelte**: 주간 제약 조건(No React)에 부합. 반응형 색상 업데이트에 Svelte의 반응성 모델이 적합하며 번들 크기도 작음
- **OKLCH**: 사람이 인지하는 색상과 가장 가까운 색공간으로, 무드를 유지하면서 접근성을 개선한 대안 팔레트 생성에 핵심
- **Vite**: 빠른 개발 서버와 최적화된 빌드

### 핵심 기술 도전
1. **색각 이상 시뮬레이션 정확도**: Brettel/Viénot 알고리즘을 선형 RGB 공간에서 적용해야 하며, 색공간 변환 순서가 정확도에 결정적. OKLCH → Linear sRGB → CVD 행렬 → sRGB 순서 유지 필요
2. **패션 맥락 접근성 점수 설계**: 웹 UI의 WCAG는 텍스트 가독성 기준이라 의류 색상에 바로 적용 불가. 명도 대비, 색조 다양성, 채도 분포를 가중 합산한 커스텀 메트릭 설계 필요
3. **대안 팔레트 탐색 알고리즘**: OKLCH 공간에서 원본 색상의 무드(hue angle 유지)를 보존하면서 접근성을 높이는 방향으로 채도/명도를 이동시키는 그리디 탐색 구현

## 🎨 무엇이 흥미로운가?

### 독특한 각도
접근성 도구는 항상 웹/UI 개발자를 대상으로 만들어졌다. 패션이라는 전혀 다른 도메인에 접근성 개념을 이식하면서, "착용 가능한 접근성(wearable accessibility)"이라는 새로운 개념을 정립한다. 패션 색상 조합이 실루엣 위에 레이어로 쌓이는 방식은 UI의 전경/배경 구조와는 다른 복잡성을 가진다.

### 혁신 포인트
- 패션 레이어(상의, 하의, 아우터)라는 도메인별 색상 관계 모델링
- OKLCH 기반 무드-보존 대안 팔레트 자동 생성 (단순 색상 교체가 아닌 색공간 탐색)
- 시즌 트렌드 팔레트에 접근성 점수를 부여하는 새로운 메타데이터
- 패션 실루엣에 적용한 색각 이상 시뮬레이션 (추상적 색상 칩이 아닌 실제 맥락)

### 학습 기회
- OKLCH, OKLAB 색공간의 수학적 원리
- Brettel/Viénot 색각 이상 시뮬레이션 알고리즘
- APCA(Advanced Perceptual Contrast Algorithm) 대비 계산
- Svelte 반응성 시스템과 실시간 색상 업데이트 패턴
- 색공간 변환 파이프라인 (감마 보정, 선형화 등)

## 📊 시장 및 검증

### 유사 프로젝트
- **Coblis Color Blindness Simulator** (color-blindness.com): 단순 이미지 업로드 후 전체 화면 시뮬레이션. 패션 레이어 개념 없음, 대안 팔레트 미제공
- **Colour Contrast Analyser** (TPGi): 두 색상 간 대비만 측정. 다층 의류 조합이나 트렌드 팔레트 없음
- **Adobe Color** (color.adobe.com): 접근성 탭이 있지만 패션 맥락 없이 UI 색상 기준 적용

### 이 간극이 존재하는 이유
접근성 도구는 전통적으로 ADA/WCAG 컴플라이언스가 법적 요구사항인 디지털 제품 팀을 위해 만들어졌다. 패션 업계는 접근성 개념에 노출이 적고, 패션 색상 조합의 복잡성(레이어, 소재 질감, 조명 등)을 기존 도구가 다루지 않는다.

### 성공 지표
- [ ] 4가지 색각 이상 시뮬레이션이 기존 연구와 시각적으로 일치
- [ ] 패션 블로거/스타일리스트 5명 이상의 실사용 피드백 수집
- [ ] "착장 접근성 점수" 계산 로직을 색각 이상 당사자에게 검증
- [ ] 대안 팔레트 생성 시 원본 색조(hue) ±15° 이내 유지

## 🌱 배울 것들

### 새로운 기술과 개념
- **색공간 수학**: OKLCH의 L(밝기), C(채도), H(색조) 축이 인간 지각과 어떻게 대응되는지 깊이 이해
- **CVD 알고리즘**: Brettel 1997, Viénot 1999 논문의 행렬 변환을 직접 구현하며 색각의 신경과학적 원리 학습
- **접근성 메트릭 설계**: WCAG 2.1의 명도 대비 공식과 APCA의 차이점, 그리고 새로운 도메인에 맞는 커스텀 메트릭 설계 방법
- **Svelte 반응성 모델**: React와 다른 Svelte의 컴파일 기반 반응성이 실시간 색상 업데이트 같은 시나리오에서 왜 우아한지

### 성장 기회
- 색공간 변환 파이프라인에서 감마 보정과 선형화가 왜 순서에 민감한지 체득
- 객관적 수치가 없는 도메인(패션 미감)에 접근성 기준을 어떻게 맥락화할지 고민하는 설계 능력
- 색각 이상 당사자 관점에서 도구를 검증하는 실제 접근성 테스트 경험

### 도전 구간
색각 이상 시뮬레이션 알고리즘은 색공간 변환 순서가 틀리면 완전히 다른 결과가 나온다. 선형 sRGB vs. 감마 인코딩된 sRGB의 차이를 실수로 경험하며 정확히 이해하게 될 것이다.
