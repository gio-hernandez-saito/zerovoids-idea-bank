# 탭 교통 흐름 시각화 확장 — 브라우저 탭을 교통망으로

> 브라우저 탭들 사이의 이동 패턴을 도시 교통망처럼 시각화해, 당신의 인터넷 탐색 습관을 한눈에 파악한다.

## 🎯 문제

### 현재 상황
현대인은 하루에 수십 개의 탭을 열고 닫으며 웹을 탐색한다. 어떤 사이트에서 어떤 사이트로 이동했는지, 어떤 탭이 '교통 허브'처럼 자주 경유되는지 전혀 알 수 없다. 브라우저는 히스토리를 순서대로 저장하지만, 탭 간 이동 패턴을 구조적으로 보여주는 도구는 존재하지 않는다.

### 불편한 점들
- **탐색 패턴 불투명**: 내가 오늘 어떤 경로로 정보를 찾았는지 복기할 방법이 없다
- **탭 과부하**: 50개의 탭이 열려 있을 때 어느 탭이 핵심이고 어느 탭이 막다른 골목인지 모른다
- **생산성 블랙홀**: 의도했던 작업에서 얼마나 멀리 벗어났는지 인식하지 못한다
- **리서치 재구성 불가**: 며칠 전 훌륭한 조사 흐름을 다시 만들어내고 싶어도 기억에 의존해야 한다

### 누가 겪는가?
하루에 수십 개 탭을 열고 복잡한 리서치를 하는 연구자, 개발자, 저널리스트, 지식 노동자들. 특히 여러 주제를 동시에 조사하거나 딥다이브 리서치를 자주 하는 사람들.

## 💡 해결책

### 핵심 개념
탭 교통 흐름 시각화 확장은 브라우저 탭 간 이동을 실시간으로 추적하여, 교통망 다이어그램처럼 시각화한다. 각 도메인(사이트)은 교차로, 탭 간 이동은 도로, 자주 경유되는 사이트는 대형 허브 역처럼 표현된다. 도시 교통 분석에서 쓰는 '흐름 두께(flow width)' 개념을 적용해 — 많이 이동한 경로일수록 굵은 선으로 표현한다.

### 작동 방식
1. **백그라운드 추적**: 확장이 탭 활성화 이벤트(chrome.tabs API)를 감청하며 이동 순서를 기록한다
2. **그래프 구축**: 도메인 노드와 이동 엣지로 방향성 그래프(directed graph)를 실시간으로 구성한다
3. **시각화 팝업**: 확장 아이콘 클릭 시 D3.js로 렌더링된 교통망 맵이 팝업으로 열린다
4. **인사이트 레이어**: 가장 많이 방문한 '허브', 막다른 골목 탭, 세션의 '중심 경로'를 하이라이트로 표시한다
5. **세션 저장**: 리서치 세션을 JSON으로 저장하고 나중에 다시 불러와 흐름을 복기할 수 있다

### 주요 기능
- **실시간 흐름 맵**: 탭을 이동할 때마다 그래프가 살아 움직이며 업데이트된다
- **허브 탐지**: 교통 중심지처럼 많은 탭에서 방문된 사이트를 크게 강조 표시한다
- **막다른 골목 표시**: 한 번 열고 다시 돌아오지 않은 탭들을 시각적으로 구분한다
- **세션 리플레이**: 저장된 세션의 탐색 흐름을 단계별로 재생할 수 있다
- **도메인 클러스터링**: 같은 도메인 내 여러 탭을 자동으로 그룹화한다

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────────────────────────────┐
│  Chrome Extension (Manifest V3)         │
│                                         │
│  ┌───────────────┐  ┌─────────────────┐ │
│  │  Background   │  │  Popup UI       │ │
│  │  Service      │  │  (D3.js Graph)  │ │
│  │  Worker       │  │                 │ │
│  │               │  │  ┌───────────┐  │ │
│  │  tab events   │─▶│  │ Flow Map  │  │ │
│  │  + graph      │  │  │ Renderer  │  │ │
│  │  builder      │  │  └───────────┘  │ │
│  └───────┬───────┘  └─────────────────┘ │
│          │                              │
│  ┌───────▼───────┐                      │
│  │ Chrome        │                      │
│  │ Storage API   │                      │
│  │ (세션 저장)    │                      │
│  └───────────────┘                      │
└─────────────────────────────────────────┘
```

### 기술 스택 선택 이유
- **Chrome Extension API (MV3)**: `chrome.tabs.onActivated`, `chrome.tabs.onCreated` 이벤트로 탭 이동을 정밀하게 추적. Manifest V3는 현재 표준이며 보안이 강화됨
- **TypeScript**: 그래프 자료구조(노드, 엣지, 가중치)를 타입 안전하게 모델링
- **D3.js**: Force-directed graph 레이아웃으로 탭 관계를 자연스럽게 시각화. 노드 크기, 엣지 두께를 데이터에 따라 동적으로 조절
- **Vite**: 확장 빌드를 위한 빠른 번들링. `vite-plugin-web-extension` 활용

### 핵심 기술적 도전
1. **그래프 레이아웃 안정성**: 탭이 추가될 때마다 force simulation이 재시작되면 맵이 요동친다. 노드 고정(pin) 전략과 점진적 업데이트가 필요하다
2. **Manifest V3 Service Worker 수명**: MV3에서 백그라운드 스크립트는 비활성 시 종료된다. `chrome.storage`를 통한 상태 영속화와 이벤트 기반 재활성화 전략을 구현해야 한다
3. **팝업 크기 제한**: 확장 팝업은 800×600px 제한이 있다. 큰 그래프를 위한 pan/zoom과 클러스터 접기(collapse) 기능이 필요하다

## 🎨 무엇이 흥미로운가?

### 독창적인 관점
브라우저 탭 관리 도구는 많지만, 탭을 '교통 노드'로 취급하는 도구는 없다. 도시 교통 분석에서 쓰는 OD(Origin-Destination) 매트릭스 개념을 브라우저 탐색 패턴에 적용하는 것이 핵심 아이디어다. 이 메타포 덕분에 직관적으로 이해할 수 있는 시각화가 가능하다.

### 혁신 포인트
- 교통 흐름 분석 알고리즘(흐름 두께, 허브 중심성)을 브라우저 탭에 적용
- 탭 이동을 방향성 그래프(directed graph)로 모델링해 탐색 패턴의 구조를 드러냄
- 단순 히스토리 나열이 아닌 공간적 네트워크 시각화로 인지 부하를 줄임
- '세션 리플레이' — 탐색 흐름을 영화처럼 재생하는 기능

### 배울 수 있는 것들
- Chrome Extension Manifest V3의 Service Worker 패턴
- D3.js Force-Directed Graph 심층 활용
- 방향성 그래프 자료구조와 중심성(centrality) 알고리즘
- 브라우저 확장의 상태 영속화 전략

- **Workona / Toby**: 탭을 그룹으로 정리하지만, 탭 간 이동 흐름은 시각화하지 않는다
- **History Timeline Extensions**: 방문 기록을 시간순으로 보여주지만 관계망은 표현하지 않는다
- **WebGraph (연구용)**: 웹 페이지 간 링크 그래프를 분석하지만 실제 사용자 탐색 패턴을 추적하지 않는다

### 이 갭이 존재하는 이유
브라우저 탭 관리는 주로 '정리(organizing)' 관점에서 접근해왔다. '분석(analyzing)' 관점, 특히 교통망 비유를 차용한 접근은 시도된 바가 없다. 또한 D3 force graph를 MV3 확장 팝업에 통합하는 것이 기술적으로 까다로워 진입장벽이 존재한다.
