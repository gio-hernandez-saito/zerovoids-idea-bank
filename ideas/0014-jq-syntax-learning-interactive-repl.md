# jq 문법 학습 인터랙티브 REPL

> jq를 '배우면서' 쓰는 CLI — 필터를 입력하면 왜 그 결과가 나왔는지 한국어로 설명해준다

## 🎯 문제

### 현재 상황
jq는 강력한 JSON 처리 CLI 도구지만, 문법이 독특해서 처음 접하는 사람은 공식 문서를 펼쳐두고 시행착오를 반복한다. `man jq`는 레퍼런스이지 튜토리얼이 아니며, 온라인 플레이그라운드(jqplay.org)는 결과만 보여줄 뿐 *왜* 그런 결과가 나왔는지 설명하지 않는다.

### 불편함
- **학습 곡선**: `.foo[]`, `select()`, `map()` 등 체이닝 문법이 직관적이지 않아 10분 작업을 1시간 삽질로 만든다
- **설명 부재**: 결과값은 얻어도 내가 쓴 필터가 어떤 원리로 동작하는지 모른 채 넘어가게 된다
- **반복 검색**: 비슷한 패턴을 매번 구글링하면서 같은 StackOverflow 답변을 읽는다

### 누가 겪는가?
JSON API를 다루는 백엔드·데브옵스 엔지니어, 로그 분석이 필요한 SRE, 쉘 스크립트를 자주 짜는 개발자. jq를 알긴 아는데 고급 필터는 아직 어색한 중급자가 핵심 타깃이다.

## 💡 해결책

### 핵심 아이디어
터미널에서 실행하는 인터랙티브 REPL로, jq 필터를 입력할 때마다 결과와 함께 **단계별 실행 설명**을 한국어(또는 영어)로 출력한다. 마치 언어 학습 앱처럼 "이 표현은 이런 의미입니다"를 옆에 붙여주는 방식이다.

### 작동 방식
1. **JSON 로드**: 파일 경로나 파이프로 JSON을 입력받는다 (`jq-learn < data.json`)
2. **필터 입력**: 프롬프트에 jq 필터를 입력한다 (예: `.users[] | select(.age > 30) | .name`)
3. **결과 + 설명 출력**: 오른쪽(또는 아래)에 필터를 AST 수준으로 분해하여 각 연산자가 무슨 역할을 하는지 단계별로 설명한다
4. **히스토리 & 북마크**: 잘 된 필터를 저장해두고 나중에 재사용하거나 치트시트로 export한다

### 주요 기능
- **단계별 실행 트레이스**: `.users[]` → "배열의 각 원소를 펼칩니다", `select(.age > 30)` → "age 필드가 30 초과인 원소만 통과시킵니다"
- **필터 자동완성**: 현재 JSON 스키마를 분석해 `.` 다음에 올 수 있는 키를 제안
- **오류 친절 설명**: jq 오류 메시지를 사람이 읽을 수 있는 말로 번역 ("null has no keys" → "이 값은 객체가 아니라 null입니다. 상위 경로를 확인해보세요")
- **치트시트 export**: 세션에서 사용한 필터 패턴을 Markdown 파일로 저장

## 🏗️ 기술 아키텍처

### 시스템 구조

```
┌─────────────────────────────────────────────┐
│  CLI Entry (Node.js / Ink)                  │
│  - JSON 입력 처리                            │
│  - 프롬프트 렌더링 (React-like TUI)          │
└───────────────────┬─────────────────────────┘
                    │
        ┌───────────▼───────────┐
        │  jq-web (WASM)        │  ← 실제 jq 실행
        │  필터 실행 & 결과 반환  │
        └───────────┬───────────┘
                    │
        ┌───────────▼───────────┐
        │  Filter Explainer     │  ← 핵심 모듈
        │  - 필터 AST 파싱       │
        │  - 단계별 설명 생성    │
        │  - 오류 메시지 번역    │
        └───────────┬───────────┘
                    │
        ┌───────────▼───────────┐
        │  Session Store        │
        │  - 히스토리 관리       │
        │  - 북마크 저장         │
        │  - Markdown export    │
        └───────────────────────┘
```

### 기술 스택 선택 이유
- **Ink (React for CLI)**: 터미널 UI를 React 컴포넌트로 구성. 좌우 분할 레이아웃(입력 | 설명)을 선언적으로 만들 수 있다
- **jq-web (WASM 빌드)**: 실제 jq 바이너리를 시스템에 설치하지 않아도 동일한 결과를 낼 수 있다. 크로스 플랫폼 보장
- **TypeScript**: 필터 AST 타입 정의, 설명 맵핑 로직을 타입 안전하게 작성

### 핵심 기술 도전
1. **필터 AST 파싱**: jq 필터 문자열을 직접 파싱해 연산자별로 쪼개야 한다. 기존 jq 파서가 없으므로 PEG.js나 직접 구현이 필요
2. **설명 맵핑 완성도**: 모든 jq 연산자를 커버하는 설명 딕셔너리를 만들어야 한다. 초기엔 자주 쓰는 30개 패턴으로 시작
