# 물리 법칙으로 배우는 미적분 시각화 도구

> 공이 포물선을 그리며 날아가는 순간, 미분·적분이 왜 존재하는지 눈으로 확인한다.

## 🎯 문제

### 현재 상황
미적분은 고등학생과 대학 신입생 대다수가 가장 어렵다고 꼽는 수학 과목이다. 교과서는 극한과 엡실론-델타 정의로 시작하지만, 학생들은 "이걸 왜 배우지?"라는 물음표를 품은 채 공식을 암기한다. 물리학이 미적분의 가장 직관적인 무대임에도 불구하고, 수학 수업과 물리 수업은 별개로 진행된다.

### 핵심 불편
- **추상성 장벽**: 도함수가 "순간 변화율"이라는 말은 이해하지만, 실제 세계에서 그것이 무엇을 의미하는지 감이 없다.
- **단방향 학습**: 교재와 인강은 공식을 설명하지만, 학생이 직접 파라미터를 바꾸고 결과를 관찰하는 방식이 아니다.
- **맥락 단절**: "왜 적분이 넓이인가?"를 설명하는 그래프는 정적인 이미지이고, 수식과 물리 현상의 연결고리를 직접 탐색할 수 없다.

### 대상 사용자
미적분을 처음 접하는 고등학생·대학 1학년, 그리고 미적분을 가르치는 선생님이 주요 사용자다. 또한 "수식 없이 직관만으로 이해하고 싶은" 성인 학습자도 해당된다.

## 💡 해결책

### 핵심 개념
물리 시뮬레이션을 조작하면서 미적분 개념을 *발견*하는 도구다. 단순히 그래프를 보여주는 것이 아니라, 사용자가 공의 초기 속도를 바꾸거나 중력 값을 조절하면 실시간으로 위치·속도·가속도 그래프가 갱신된다. "미분 = 기울기"가 아니라, "지금 이 공이 얼마나 빠르게 움직이는가"라는 질문에 답하는 과정에서 미분이 등장한다.

### 학습 흐름
1. **시나리오 선택**: 포물선 운동, 단진동, 자유낙하, 용수철 등 물리 시나리오를 고른다.
2. **파라미터 조작**: 슬라이더로 초기 조건(각도, 속도, 질량 등)을 실시간으로 변경한다.
3. **그래프 연동 관찰**: 시뮬레이션 애니메이션 옆에 위치(x), 속도(v), 가속도(a) 그래프가 동기화되어 나타난다. 공이 최고점에 도달할 때 속도 그래프가 0을 지나는 순간을 직접 목격한다.
4. **개념 잠금 해제**: 특정 조건을 달성하면 "지금 본 것이 미분의 정의입니다" 같은 짧은 설명이 팝업된다. 수식은 이미 경험한 것의 언어로 등장한다.
5. **적분 모드**: 속도 그래프 아래 넓이를 색칠하면서 "이것이 이동 거리"임을 리만 합 애니메이션으로 확인한다.

### 주요 기능
- **시나리오별 탐색 모드**: 포물선·단진동·유체저항 등 5개 이상의 물리 시나리오, 각각 다른 미적분 개념과 연결
- **실시간 3-패널 뷰**: 시뮬레이션 / 수식 / 그래프가 동시에 반응
- **리만 합 슬라이더**: 분할 개수 n을 1~1000 사이에서 조절하며 적분의 근사가 수렴하는 과정을 직접 체험
- **"왜?" 버튼**: 어느 순간에나 누르면 해당 상태에 맞는 짧은 직관적 설명이 표시됨

## 🏗️ 기술 아키텍처

### 시스템 개요

```
┌─────────────────────────────────────────────┐
│              UI Layer (React)               │
│  [시뮬레이션 캔버스] [수식 패널] [그래프 패널] │
└───────────────┬─────────────────────────────┘
                │
┌───────────────▼─────────────────────────────┐
│          State Manager (Zustand)            │
│  파라미터 상태 / 시간 스텝 / 계산 결과 공유   │
└──────┬─────────────────────┬────────────────┘
       │                     │
┌──────▼──────┐     ┌────────▼────────┐
│  Physics    │     │  Math Engine    │
│  (Matter.js)│     │  (수치 미분·적분) │
│  시뮬레이션  │     │  D3 그래프 렌더  │
└─────────────┘     └─────────────────┘
```

### 기술 스택 선택 이유
- **Matter.js**: 브라우저에서 동작하는 2D 물리 엔진. 중력, 충돌, 탄성 등을 별도 구현 없이 제공하며, 시뮬레이션 루프가 D3 업데이트 주기와 동기화하기 쉽다.
- **D3.js**: 위치·속도·가속도 시계열 그래프, 리만 합 영역 색칠, 접선 애니메이션 등 수학적 시각화에 최적화.
- **React + TypeScript**: 파라미터 슬라이더와 시나리오 선택 UI를 컴포넌트로 분리하여 시나리오 추가가 쉬움.
- **Zustand**: 물리 엔진의 시간 스텝 데이터를 그래프와 수식 패널이 구독하는 간결한 상태 관리.

### 핵심 기술 과제
1. **물리 루프 ↔ D3 동기화**: Matter.js의 `afterUpdate` 이벤트에서 위치·속도를 샘플링해 D3 데이터 배열에 push하고, `requestAnimationFrame`으로 그래프를 갱신하는 타이밍 조율이 필요하다.
2. **수치 미분의 정확도 vs 가시성 트레이드오프**: 너무 정밀한 수치 미분은 그래프가 너무 매끄러워 교육적 효과가 떨어진다. 의도적으로 유한 차분(finite difference)을 작은 h값 단계별로 보여주는 "미분 정밀도 슬라이더"가 필요하다.

## 🎨 왜 흥미로운가?

### 차별화 포인트
대부분의 수학 교육 도구는 함수 그래프를 그려주는 수준이다(GeoGebra, Desmos). 이 도구는 반대 방향에서 출발한다: 물리 세계를 조작하면 수학이 뒤따라온다. "함수를 주면 그래프를 보여준다"가 아니라 "현상을 보면 함수가 나타난다".

### 혁신 포인트
- 물리 시뮬레이션을 미적분 개념 발견의 *실험 도구*로 사용
- 리만 합의 수렴을 인터랙티브 슬라이더로 직접 체험
- 수식이 경험 이후에 등장하는 구성 순서 (개념 → 언어 순서)
- "왜?" 버튼으로 맥락 민감형 설명 제공

### 학습 기회
- Matter.js 물리 엔진 통합 및 커스텀 시뮬레이션 설계
- D3.js 실시간 데이터 스트림 시각화
- 수치 해석 기초 (유한 차분법, 리만 합)
- 교육 UX 설계 원칙 (스캐폴딩, 점진적 공개)

## 📊 시장 및 검증

### 유사 프로젝트
- **Desmos** (desmos.com): 함수 그래프 도구. 물리 시뮬레이션과 연동되지 않으며, 학습자가 수식을 먼저 입력해야 한다.
- **GeoGebra** (geogebra.org): 강력하지만 인터페이스가 복잡하고, 물리 현상 → 수학 개념의 흐름이 아닌 수학 → 시각화 방향.
- **3Blue1Brown "Essence of Calculus"**: 유튜브 영상으로 탁월하지만 수동적 시청이다. 파라미터를 직접 조작할 수 없다.

### 이 공백이 존재하는 이유
물리 시뮬레이션 엔진과 수학 시각화 라이브러리를 조합해 "개념 발견" 플로우로 엮는 것은 구현 복잡도가 높다. 교육 앱 개발자는 주로 콘텐츠 제작에 집중하고, 프론트엔드 개발자는 교육학적 설계를 간과한다.
